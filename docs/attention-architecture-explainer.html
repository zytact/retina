Plain-language model briefing

How the model learns what to look at.

This page explains attention, self-attention, cross-attention, ConvNeXt-Tiny, ImageNet pretraining, OCTA-SimCLR adaptation, F_octa, and z-scores in the exact context of the planned Retina CAD model.

Attention is weighted relevance.

The model creates a score for each source of information, turns scores into weights, then uses those weights to make a mixture of the most relevant information for the current patient.

Attention(Q, K, V) = softmax(QKᵀ / √d) · V

Attention, from first principle.

Attention is not a separate kind of knowledge. It is a routing mechanism. It lets one representation ask which other representations matter right now, then pull information from them in proportion to that relevance.

Attention itself

Think of a query asking a question. Keys describe what each available item contains. Values are the actual information to retrieve. The query is compared with every key. Better matches get larger weights. The output is a weighted sum of the values.

In this model, attention helps avoid a fixed rule like “always trust the superficial layer most.” Instead, the model can learn that the deep capillary layer, choriocapillaris, or clinical context matters more for a specific patient pattern.

Self-attention

Self-attention means the query, keys, and values all come from the same set of tokens. Here, the set is the three OCTA layer tokens: superficial, deep, and choriocapillaris.

It asks: “within the OCTA layers, how should each layer read the other layers?” For example, a superficial vascular feature can be interpreted differently if the deep layer also shows rarefaction.

Cross-attention

Cross-attention means the query comes from one source, while keys and values come from another. In the planned fusion block, F_octa asks a question, and F_clinical supplies clinical context as keys and values.

It asks: “given this OCTA pattern, which clinical facts should change the interpretation?” The same retinal finding may matter differently in a diabetic, hypertensive, or normotensive patient.

How many layers are there?

“Layer” means different things depending on context. The model has anatomical OCTA layers, neural network layers, attention blocks, and training stages. Mixing those meanings is the usual source of confusion.

3 OCTA layers Superficial, deep, choriocapillaris. These are retinal depth slabs, not neural network layers.
3 backbones One ConvNeXt-Tiny encoder per OCTA layer. They do not share weights, because each vascular layer has different texture.
4 ConvNeXt stages ConvNeXt-Tiny internally uses four feature stages with depths 3, 3, 9, 3 and output channels 96, 192, 384, 768.
1 self-attention block Eight heads read interactions among the three OCTA tokens, then mean-pooling creates F_octa.
1 cross-attention block Four heads let the OCTA representation query clinical metadata.
Several MLP layers Projection heads reduce 768 to 256, clinical metadata becomes 256, and the classifier maps the final 512 representation to disease classes.

Term sheet.

Short answers for the architecture vocabulary used in the plan. These are written for reading the notebook and model code later.

What is ConvNeXt-Tiny?

A modern convolutional neural network. “Tiny” is the smallest common ConvNeXt size, not tiny in capability. It turns an image into increasingly abstract feature maps, ending with a 768-dimensional visual representation.

Why is it the backbone?

The backbone is the main feature extractor. It does the heavy image understanding before smaller heads perform projection, attention, fusion, and classification.

What does pretrained on ImageNet mean?

The ConvNeXt weights start from training on ImageNet, a large natural-image dataset. That gives useful low-level filters and general visual structure before the model sees OCTA.

What is OCTA-SimCLR?

SimCLR is self-supervised contrastive learning. OCTA-SimCLR means using OCTA images to teach the backbone which retinal images should be close in representation space without needing CAD labels.

What does adapt mean here?

Adapt means taking ImageNet-initialized ConvNeXt and continuing self-supervised training on OCTA. The features shift from natural photographs toward grayscale vessel and perfusion patterns.

What is F_octa?

F_octa is the 256-dimensional summary after the model attends across superficial, deep, and choriocapillaris tokens. It is the layer-aware OCTA representation.

What are z-scores?

A z-score says how far a value is from the training-set mean in units of standard deviation: z = (x − mean) / std. It makes age, BMI, and image intensities comparable in scale.

Why use z-scores in this model?

Neural networks train more stably when inputs have similar scale. The plan computes mean and standard deviation on the training fold only, then applies those statistics to validation and test data to avoid leakage.

The full path

Sup, Deep, and CC images each pass through their own ConvNeXt-Tiny backbone. Their 768-dimensional outputs are projected to 256-dimensional tokens. Layer-identity embeddings mark which token is which. Self-attention creates F_octa. Clinical metadata becomes F_clinical. Cross-attention fuses OCTA with clinical context. The final 512-dimensional representation goes to the classifier.

Shape summary.

The dimensions are not arbitrary. They control capacity, compute, and overfitting risk for a medium-sized medical dataset.
