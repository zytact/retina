Phase 2 — Data Preparation

From Raw Files to Ready Tensors

Raw OCTA images and clinical spreadsheets are not directly usable by a neural network. Phase 2 transforms them into clean, normalized, augmentation-ready tensors while preventing the most common source of data leakage: having the same patient's two eyes end up in different cross-validation folds.

Key Principle

Every preprocessing decision that uses information from the dataset (e.g., computing the mean pixel value) must be fit on the training fold only and then applied to the validation and test folds. Otherwise you leak information about the held-out data into the training process.

The Big Picture: What Phase 2 Produces

Phase 2 takes the raw RASTA files on disk and produces a PyTorch Dataset object that the rest of the pipeline (SimCLR pretraining, classifier training) can consume directly. Each "sample" in the dataset is a dictionary containing:

  • The three OCTA layer images (Sup, Deep, CC) as 1-channel 224x224 tensors
  • Clinical features as a flat vector + a mask indicating which values were observed
  • Placeholder tensors for segmentation masks (filled in later during Phase 8)
  • The disease label, patient ID, eye side (OD/OS), and fold assignment
Beginners: A "Dataset" in PyTorch is just a Python object with a __len__ and a __getitem__ method. When you index it like dataset[5], it loads the 6th sample from disk, processes it, and returns ready-to-use tensors. This is the standard way to feed data into a PyTorch training loop.

Bilateral Eye Handling (Critical Rule)

This is one of the most important design decisions in the entire project, and getting it wrong invalidates the results.

The RASTA dataset contains patients with one or both eyes scanned. A patient's two eyes are not independent -- they share genetics, systemic circulation, age, and disease status. If we put OD in the training set and OS in the test set, the model has effectively seen that patient's disease pattern during training and will score artificially well on the test set. This is a subtle form of data leakage that inflates AUC and fools nobody once detected.

The rule is simple and absolute: All eyes from the same patient must be assigned to the same cross-validation fold.

In practice, this means we group the manifest by patient ID before splitting, then apply stratified k-fold at the patient level (stratifying by cohort label and optionally age decade). After splitting, we verify that no patient ID appears in more than one fold.

OCTA Image Preprocessing (Step by Step)

Each OCTA image goes through a deterministic pipeline. The goal is to standardize the images for neural network consumption while preserving the vascular structure that carries disease information.

Step 1: Quality Control

We check each image for basic usability: is it all black? Is there zero variance (a dead pixel)? Is the signal-to-noise ratio too low? Optionally, a lightweight CNN quality classifier can flag scans with motion artifacts. Low-quality scans are excluded and logged. This prevents the network from learning to recognize artifacts instead of anatomy.

Step 2: Percentile Normalization

For each image independently, we compute the 2nd and 98th percentiles of pixel intensity, then clip and rescale the image to [0, 1]. This removes extreme pixel outliers (e.g., a single bright reflection) without assuming the data is Gaussian. Unlike global min-max normalization, percentile clipping is robust to a few spurious bright or dark pixels.

Step 3: CLAHE (Contrast-Limited Adaptive Histogram Equalization)

OCTA images often have uneven illumination -- the center might be brighter than the edges. CLAHE divides the image into an 8x8 grid of tiles, equalizes each tile's histogram independently, and then blends the tile boundaries to avoid artifacts. The "contrast-limited" part means it caps how much contrast can be amplified, preventing noise amplification in flat regions. We apply CLAHE per layer (not globally) because each layer has different contrast characteristics.

Step 4: Resize to 224x224

All OCTA images are resized to 224x224 pixels using bicubic interpolation. This is the standard input size for ConvNeXt-Tiny (our backbone). Bicubic interpolation preserves smooth vascular boundaries better than nearest-neighbor (which creates jaggies) or bilinear (which is slightly blurrier).

Step 5: Z-Score Normalization

Finally, we subtract the per-channel mean and divide by the per-channel standard deviation. Crucially, the mean and std are computed from the training fold only and then applied to the validation and test folds. This ensures no information about the held-out data leaks into the training process. After this step, pixel values are centered around zero with unit variance, which helps neural network training stability.

Watch out: Do not compute z-score statistics on the entire dataset before splitting. Always compute on the training fold and apply to val/test. This is a common mistake that subtly leaks information.

Clinical Metadata Preprocessing

The clinical data contains a mix of continuous (age, BMI), binary (sex, hypertension, diabetes), and ordinal (smoking: never/former/current) features. Each type needs different handling.

Continuous Features

Age and BMI are z-score normalized using training-fold mean and std, just like the images. This puts them on a comparable scale.

Binary Features

Sex, hypertension, diabetes, and dyslipidemia are encoded as 0/1 integers. They are not normalized because they are already on a meaningful scale (1 means "present").

Ordinal Features (Smoking)

Smoking status has a natural order: never (0) < former (1) < current (2). Encoding these as raw integers (0, 1, 2) would imply that "current" is twice as "intense" as "former," which is not necessarily true. Instead, we use learned ordinal embeddings -- the model learns a scalar representation for each smoking category during training.

Missing Value Handling (The Clever Part)

Traditional approaches fill missing values with the mean or median of that feature. But this is problematic: imputing the mean assumes the missing value is "average," which may be systematically wrong if missingness correlates with disease status (e.g., sicker patients might be less likely to have BMI recorded).

Our approach: each clinical feature gets a learnable mask token -- a single trainable scalar that represents "this value was not measured." When a feature is missing, the model uses the mask token instead. During training, we randomly hide 10% of observed values to teach the model to handle missing data robustly. This way, the model can learn that missingness itself might be informative, rather than pretending missing values are average.

Feature Selection

Not every clinical feature is informative for disease classification. We compute mutual information (MI) between each feature and the disease label on the training fold. Features with MI < 0.01 are flagged as potentially low-information. This is a diagnostic, not an automatic filter -- domain knowledge may overrule it.

Patient- Stratified Cross-Validation Folds

We create 5 folds for cross-validation. The process is:

  1. Group all eye samples by patient ID.
  2. Stratify by disease cohort and (optionally) age decade to ensure each fold has a similar distribution of classes and ages.
  3. Assign entire patient groups to folds, never splitting a patient across folds.
  4. Verify that no patient ID appears in more than one fold.

The result is 5 approximately balanced folds, each containing 94-98 eye samples, with zero patient overlap. These fixed folds are used consistently throughout the entire project -- every phase (SimCLR, classifier training, baselines, ablations) uses the same splits.

Glossary for Beginners

CLAHE
Contrast-Limited Adaptive Histogram Equalization. A method that improves local contrast in images by equalizing histograms over small tiles, then blending them. Think of it as "smart brightness/contrast adjustment" that adapts to different regions of the image.
Z-Score Normalization
Subtracting the mean and dividing by the standard deviation. Transforms data to have mean=0 and variance=1. Also called "standardization."
Bicubic Interpolation
A method for resizing images that considers the 16 nearest pixels (a 4x4 grid) to compute each output pixel. It produces smoother results than simpler methods.
Mutual Information (MI)
A measure of how much knowing one variable tells you about another. High MI between a clinical feature and the disease label means that feature is informative for classification. MI can capture non-linear relationships, unlike correlation.
Stratified K-Fold
A cross-validation technique where each fold preserves the same class proportions as the full dataset. Prevents a fold from accidentally having zero samples of a rare class.
