Fusion means the model learns from images, generated masks, biomarkers, and clinical context together.

In this project, fusion does not mean dumping a spreadsheet next to pixels and hoping for the best. The updated plan first learns compact retinal-layer features, generates weak segmentation masks and OCTA biomarkers, encodes clinical variables plus biomarkers as tabular context, then merges those learned features into one shared pathway that produces class probabilities, uncertainty, and explanations.

What goes in

Three OCTA layers per eye, generated vessel/FAZ/flow-deficit pseudo-masks, OCTA biomarkers, plus age, sex, BMI, smoking, hypertension, diabetes, dyslipidemia, and related clinical fields.

Where fusion happens

After the image, generated-mask, and tabular branches create learned feature vectors. This is feature-level fusion, also called intermediate fusion.

Why it exists

The same retinal pattern can mean different things depending on patient context. Clinical data and generated biomarkers help the model interpret what the image pattern likely means.

Short answer

The implementation has three evidence streams. One reads the three OCTA images, one generates/encodes segmentation masks and biomarkers (via U-Net++ student), and one reads clinical variables plus those biomarkers as tabular context. These streams meet in fusion, then continue through a prototype-augmented classifier with uncertainty, calibration, and explainability.

Overview

What “fusion” means in plain language

Imagine a clinician looking at a scan and a chart at the same time. The scan shows what the eye looks like. The chart shows who the patient is and what risk factors they carry. Fusion is the model version of that joint reasoning, now with an additional mask/biomarker summary of retinal vasculature.

In machine learning terms, each input type is first encoded into a numerical summary called a feature vector. Those vectors are then combined into a shared representation that is easier for the classifier to use than raw inputs.

This project is aiming for intermediate multimodal fusion. Raw pixels stay in the image branch. Generated masks stay in the mask branch. Clinical values and generated biomarkers stay in the tabular branch. Fusion happens after modality-specific encoders have transformed them into compact learned representations.

The model is trying to classify disease cohorts from retinal OCTA. The hypothesis is that disease signal is spread across retinal depth, vascular biomarkers, generated masks, and patient context—not confined to one image or one spreadsheet field.

Image only

Useful when the scan alone contains enough signal. Weak when the same visual pattern appears in different clinical settings.

Clinical/tabular only

Useful for broad risk estimation. Now also can include generated biomarkers, but still misses full retinal texture if images are removed.

Fused model

Lets the model ask, “given this retinal pattern and this patient context, which disease cohort is most plausible?”

One sentence summary

Fusion is the step where the model turns “what the eye looks like,” “what the generated vascular masks quantify,” and “what we know about the patient” into one joint decision.

Step 1, inputs

The model sees three OCTA views of the same eye: superficial vascular complex, deep vascular complex, and choriocapillaris. It also generates pseudo-masks from those images and sees clinical variables from the spreadsheet.

These are different kinds of evidence. Images show structure. Pseudo-masks isolate vessels/FAZ/flow-deficit regions. Clinical variables show patient context.

The updated dataset object should return each layer separately, generated masks, generated biomarkers, clinical_features, and clinical_mask. Current notebook placeholders must be replaced before classifier work.

Step 2, image branch

Each retinal layer gets its own encoder. That matters because SVC, DVC, and CC do not look the same biologically or visually.

After each encoder, the model has one 256-dimensional token per layer. A cross-layer attention block then learns how the layers relate to each other.

This is already one kind of fusion: image-with-image fusion across retinal depth. The plan calls this F_octa, the layer-aware OCTA representation.

Step 3, mask + tabular branches

The mask branch uses self-generated vessel, FAZ, and CC flow-deficit pseudo-masks. The tabular branch converts clinical variables plus generated biomarkers into one compact representation.

Missing clinical values are not filled with a simple average. Failed biomarker QC is also marked explicitly.

Because RASTA has no manual segmentation labels, masks are weak pseudo-labels from classical image processing, optionally refined by a frozen segmentation student.

Step 4, fusion

This is the key point. The retinal representation, mask representation, and tabular representation are merged into one shared state before classification.

The plan uses cross-attention, where the OCTA representation is conditioned by the tabular representation. In plain language, the model asks how the retinal evidence should be read for this specific patient context and generated biomarker profile.

Because the plan compresses clinical data plus biomarkers into a single 256-d tabular token, this cross-attention behaves more like learned conditioning than full token-by-token attention. The mask feature is fused separately as structured vascular evidence.

Step 5, output

The fused representation goes to the prototype-augmented classifier head. That head predicts disease cohort probabilities with learned class prototypes that improve rare-cohort generalisation.

The output also includes MC-Dropout uncertainty (epistemic variance and entropy), post-hoc temperature calibration (ECE, Brier score), and full explainability (Grad-CAM per layer, layer-dominance analysis from attention weights, SHAP/occlusion for clinical features).

Fusion choices

Why this project uses intermediate fusion

There are three common ways to combine modalities. This project intentionally picks the middle option.

“Intermediate” means separate learning first, joint reasoning second. For this dataset, that is usually the safest compromise between flexibility, interpretability, and data efficiency.
Fusion style What it does Why it is weaker or stronger here
Early fusion Combine raw inputs very early. Hard here because pixels and clinical values are very different data types.
Late fusion Let each modality predict on its own, then average or vote. Easy to build, but weaker at learning nuanced interactions between modalities.
Intermediate fusion Learn modality-specific features first, then combine the learned features. This is the implemented design. The image branch stays image-aware, the mask branch encodes vascular structure, and the tabular branch provides clinical context before fusion.
Practical effect The tabular branch does not need to learn edge detectors. The image branch does not need to interpret missing spreadsheet fields directly. The mask branch gives explicit vessel/FAZ/flow-deficit structure. Each branch solves the part of the problem it is best suited for, then the fusion block models cross-modal dependence.
Deep dive

Architecture overview, from basic to advanced

The same architecture can be understood at several levels. Start with the short explanations. Open the advanced notes when you want the design logic.

The project goal is not only prediction accuracy. It also aims for uncertainty, calibration, and explainability that clinicians can inspect.

1. Separate encoders for the three retinal layers

Each OCTA layer shows a different depth of retinal vasculature. So each gets its own image encoder.

The backbone is ConvNeXt-Tiny, pretrained on ImageNet, then adapted with OCTA SimCLR. After pooling, each branch produces a 256-d token.

Why not share one encoder across all layers?

The plan argues that the layers reflect different physiological compartments and different textures. Shared weights may force one filter bank to fit all three patterns, which can wash out layer-specific signal.

2. Cross-layer attention before multimodal fusion

Before mixing in the clinical table, the model first asks how the three retinal layers relate to each other.

Three layer tokens enter a self-attention block with learnable layer-identity embeddings. The output is pooled into one OCTA summary vector, F_octa.

Why layer-identity embeddings matter

Without them, self-attention sees three anonymous tokens. The model would know there are three things, but not reliably know which one is SVC, DVC, or CC.

3. Generated mask and biomarker branch

Because there are no manual masks, the project creates weak vessel, FAZ, and flow-deficit masks itself.

Classical image processing makes pseudo-masks first. If they are noisy, a frozen Attention U-Net or U-Net++ student can learn to imitate them and produce cleaner standardized masks.

Why generated masks matter

They give the classifier explicit vascular structure and interpretable biomarkers such as vessel density, FAZ area, and CC flow-deficit percentage.

4. Tabular encoder with explicit missingness handling

The model does not want a blank cell or failed biomarker to quietly become fake average data.

Clinical features and generated biomarkers are z-scored fold-wise where continuous, with missingness/QC carried forward via masks or learned tokens.

Why that matters clinically

Missing data is often not random. A test that was not measured, or a biomarker that failed mask QC, can itself carry information about reliability.

5. Multimodal fusion block

This is where image evidence, mask evidence, biomarkers, and clinical evidence become one shared judgment.

The plan uses cross-attention from F_octa to F_tabular, then concatenates with F_mask and projects the result into a shared 512-d state, H.

Why use attention instead of simple concatenation?

Concatenation always gives each modality the same structural role. Attention lets the model condition image interpretation on patient context and biomarker profile more selectively.

6. Classification, prototypes, uncertainty, explainability

The final layer predicts the disease cohort and should show what evidence mattered.

The classifier head is paired with prototype logits, MC dropout, calibration, Grad-CAM per image branch, attention analysis for layer dominance, SHAP for tabular attribution, and generated-mask overlays.

Why this matters for trust

If the model says “this class is likely because the deep layer pattern mattered and diabetes strongly shifted interpretation,” that is more reviewable than a raw probability alone.

Advanced nuance

There are three fusion moments in the architecture. First, cross-layer attention across retinal layers creates F_octa. Second, generated masks pass through the U-Net++ student and mask encoder to produce F_mask and biomarkers. Third, multimodal fusion combines F_octa (conditioned on F_tabular) with F_mask.

Why this architecture exists

What we are trying to create, and why this design is sensible

The architecture follows the biology, the data format, and the practical limits of a medium-sized medical dataset.

Put simply: keep the parts separate when they are fundamentally different, then merge them when the model has something meaningful to merge.

Reason 1, retinal depth matters

SVC, DVC, and CC capture different vascular compartments. Disease may leave a stronger signal in one layer than another, or in the relationship between layers.

Reason 2, generated masks make vascular evidence explicit

Vessel density, FAZ geometry, and CC flow-deficit burden are clinically interpretable summaries that complement learned image features.

Reason 3, context changes interpretation

A retinal pattern in a diabetic patient may mean something different from a visually similar pattern in a patient without diabetes or hypertension.

Reason 4, missingness and mask QC must be handled honestly

Clinical tables are rarely perfect, and generated masks can fail QC. The architecture is designed so absent values and unreliable biomarkers do not silently become misleading pseudo-measurements.

Glossary

Core terms, translated

If you know only basic AI/ML, these are the terms that matter most on this page.

The most important mental model: an encoder turns raw data into a compact summary that later parts of the model can reason over more easily.
Feature
A learned numerical pattern the model thinks is useful, such as vessel density texture or a compact summary of age and risk factors.
Encoder
The part of the model that reads an input and converts it into a feature vector.
Token
A vector treated as one item in attention. In this project, each retinal layer becomes one token after projection.
Attention
A mechanism that learns which other tokens matter when interpreting one token.
Cross-attention
Attention where one modality is conditioned by another modality. Here, retinal evidence is conditioned by tabular context: clinical variables plus generated biomarkers.
Fusion
The step where separate sources of evidence—images, generated masks, biomarkers, and clinical variables—are merged into one shared representation used for prediction.
Representation
The internal numerical summary the model carries forward after encoding.
Pseudo-mask
An automatically generated weak segmentation mask. It is useful, but not the same as a human-drawn ground-truth mask.
Segmentation student
A frozen upstream U-Net-style model trained to imitate classical pseudo-masks and produce standardized masks for the classifier.
Biomarker
A quantitative OCTA measurement extracted from masks, such as vessel density, FAZ area, or CC flow-deficit percentage.
Calibration
How well predicted probabilities match real-world correctness. A calibrated 80% confidence score should be right about 80% of the time.
Final takeaway

The architecture story in one paragraph

We are building a multimodal OCTA classifier that first learns each retinal layer on its own, generates weak vessel/FAZ/flow-deficit masks and biomarkers, learns how the layers relate, then uses tabular context to refine retinal interpretation, and finally produces a disease prediction with uncertainty and explainability. The key idea is that retinal images, generated masks, biomarkers, and clinical variables should stay separate long enough to be understood well, then join before the final decision.

Repo truth today: the groundwork is implemented through Phase 3. Generated masks, segmentation-student refinement, and the actual fusion/classifier blocks are the next major architecture steps.
