Start from zero.

These words appear throughout the notebook and plan. Read this section first if the pipeline feels abstract.

OCTA

Optical Coherence Tomography Angiography. It is an eye scan that maps retinal blood-flow signal without dye injection. Bright linear patterns often correspond to vessels or flow.

Sup, Deep, CC

Three retinal vascular compartments. Sup is the superficial vascular complex, Deep is the deeper capillary plexus, and CC is the choriocapillaris flow layer.

Tensor

A numeric array. A grayscale OCTA image resized to 224 by 224 pixels is represented as [1, 224, 224]. A batch of 32 is [32, 1, 224, 224].

Encoder

A neural network module that converts raw input into a compact vector. Here, an image encoder converts an OCTA image into learned features.

Mask

A binary image. White pixels mark a structure, black pixels mark background. Vessel masks mark vessels. FAZ masks mark the foveal avascular zone.

Biomarker

A numeric measurement derived from biology or images. Example: vessel density is the fraction of image pixels marked as vessel.

Phase 4. Image encoders.

Turns each OCTA layer into a learned feature vector while preserving spatial maps for later explanation.

Beginner view

A raw OCTA image has 50,176 pixels after resizing. The classifier should not reason over raw pixels directly. An encoder compresses the image into a smaller, more meaningful summary.

Analogy: a clinician looks at a scan and forms a structured impression, such as sparse vessels, enlarged central avascular zone, or abnormal CC texture. The encoder learns a numeric version of that impression.

Input and output

[1, 224, 224] image in. 256 dimensional token out, one token per retinal layer.

Sup image  → ConvNeXt-Tiny Sup encoder  → 768-d pooled vector → projection → 256-d token
Deep image → ConvNeXt-Tiny Deep encoder → 768-d pooled vector → projection → 256-d token
CC image   → ConvNeXt-Tiny CC encoder   → 768-d pooled vector → projection → 256-d token

Projection head:
Linear(768 → 512) → GELU → LayerNorm
Linear(512 → 256) → LayerNorm

Why separate encoders?

LayerReason
SupContains larger superficial vessels plus capillary texture.
DeepContains finer capillary patterns and different disease sensitivity.
CCRepresents perfusion texture and flow deficits, not a normal vessel tree.
Advanced detail: Phase 4 retains final spatial feature maps before global pooling. Grad-CAM later uses these maps to show which image regions influenced prediction.

Phase 5. Tabular metadata encoder.

Converts clinical variables and generated OCTA biomarkers into a 256 dimensional context vector.

Why tabular data matters

The same OCTA pattern can mean different things depending on age, diabetes, hypertension, BMI, sex, smoking, and other clinical factors. Phase 5 gives the model that context.

Missing values are learned, not filled in

Medical spreadsheets often have missing values. Instead of mean or median imputation, the model uses a learned missing token per feature. This lets it learn the meaning of absence.

Z-score normalization

For continuous values such as age and BMI:

z = (value − training_mean) / training_std

Use training-fold statistics only, otherwise validation data leaks into training.

Feature typeExampleHandling
ContinuousAge, BMIFold-fitted z-score normalization.
BinaryHypertension, diabetes0 or 1 with explicit missingness mask.
OrdinalSmoking statusLearned ordinal representation.
BiomarkerVessel density, FAZ areaGenerated in Phase 8, normalized per fold, appended to clinical stream.
Input: clinical features + Phase 8 biomarkers
LayerNorm(D_tab)
Linear(D_tab → 128) → GELU → Dropout(0.3)
Linear(128 → 256) → GELU → LayerNorm → Dropout(0.2)
Output: F_tabular ∈ R^256

Phase 6. Cross-layer attention.

Lets Sup, Deep, and CC feature tokens compare themselves before the model sees clinical metadata.

Beginner view

Attention is a learned weighting mechanism. It asks: when interpreting this layer, which other layer should matter?

A clinician does this naturally. DVC rarefaction may be interpreted differently if SVC and CC are also abnormal. Phase 6 makes that comparison learnable.

Layer identity matters

Without identity embeddings, the transformer receives three anonymous vectors. Identity embeddings tell it which token is Sup, Deep, and CC.

T_sup  = Fs + e_sup
T_deep = Fd + e_deep
T_cc   = Fc + e_cc

Input sequence: [T_sup, T_deep, T_cc] ∈ R^(3×256)
Transformer encoder:
  pre-norm multi-head self-attention
  8 heads, 32 dimensions per head
  feed-forward MLP: 256 → 1024 → 256
  GELU, dropout 0.1
Mean pool over 3 tokens
Output: F_octa ∈ R^256
Explainability: attention weights can be summarized into layer-dominance scores, showing whether a cohort tends to rely more on Sup, Deep, or CC.

Phase 7. Multimodal fusion.

Combines image evidence, clinical context, generated biomarkers, and generated mask features.

Why fusion is not just concatenation

Concatenation treats all modalities as equally relevant. Cross-attention lets OCTA features selectively use clinical context. A vascular pattern in a diabetic patient may not mean the same thing in a non-diabetic patient.

Three vectors enter fusion

  • F_octa, image-layer representation.
  • F_tabular, clinical plus biomarker representation.
  • F_mask, generated-mask representation.
Q = Linear(F_octa)
K = Linear(F_tabular)
V = Linear(F_tabular)
Attended = softmax(QKᵀ / √256) · V
F_fused = LayerNorm(F_octa + Attended)

Mask branch input: [B, 5, 224, 224]
Channels:
  1. Sup vessel mask
  2. Deep vessel mask
  3. Sup FAZ mask
  4. Deep FAZ mask
  5. CC flow-deficit mask

Mask encoder → F_mask ∈ R^128
Final projection → H ∈ R^512

Classical pseudo-mask, defined plainly

A classical pseudo-mask is made by fixed image-processing rules. It is not a neural network and has no learned weights.

OCTA image → filters → thresholds → morphology cleanup → binary mask

Sup and Deep vessel rules

1. Percentile normalize image intensity
2. Apply CLAHE contrast enhancement
3. Mild Gaussian smoothing
4. Build vessel score:
     0.55 × Frangi/Sato vesselness
   + 0.30 × local bright-vessel contrast
   + 0.15 × top-hat bright-vessel filter
5. Try local, adaptive, and percentile thresholds
6. Pick candidate with plausible vessel density
7. Light morphology cleanup
8. Suppress bottom-right watermark region
9. Skeletonize for vessel-length biomarker
RulePurpose
CLAHEImproves local contrast so faint vessels become more visible.
Frangi or Sato vesselnessResponds to line-like and tube-like structures.
Top-hat filterHighlights bright small structures against local background.
ThresholdingConverts continuous vessel score into black and white pixels.
MorphologyRemoves tiny artifacts and lightly repairs broken fragments.
SkeletonizationReduces vessels to one-pixel centerlines for length density.

FAZ rules

Vessel mask + low-flow map
→ central search window
→ connected avascular components
→ choose plausible component nearest center
→ fill and smooth
→ QC by area and centroid distance

CC flow-deficit rules

CC image
→ normalize + CLAHE
→ local background correction
→ Sauvola/adaptive dark threshold
→ morphology cleanup
→ QC by area fraction

A montage is a human review sheet. It puts original Sup, Deep, and CC images beside the generated masks for the same eye. It is for visual QC, not model training.

Ground truth versus pseudo-labels.

This distinction controls what claims are scientifically valid.

TermMeaning in this project
Classification ground truthThe disease cohort label from the dataset folder or clinical table. This is used to train and evaluate disease classification.
Segmentation ground truthExpert-drawn vessel, FAZ, or CC flow-deficit masks. These are not currently available.
Pseudo-labelAn automatic label generated by rules or another model. Phase 8 masks are pseudo-labels.
Student targetA QC-passed improved pseudo-mask used to train U-Net++ if the student route is selected.
Paper-safe wording: use generated pseudo-masks, weak masks, or automatically generated masks. Do not write ground-truth segmentations unless expert annotations are created.

U-Net++ is a decision, not a default skip.

U-Net++ is a learned segmentation architecture. It can regularize pseudo-masks, but it cannot invent ground truth.

If trained now, U-Net++ would learn from Phase 8 pseudo-masks. That means the pseudo-labels must be anatomically plausible first. A student model trained on bad labels learns bad labels.
