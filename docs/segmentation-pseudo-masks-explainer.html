Plan update: self-generated segmentation, not manual annotation

We generate the masks ourselves.

RASTA does not ship with hand-drawn vessel, FAZ, or choriocapillaris masks. The segmentation branch therefore starts as deterministic pseudo-mask generation, may add a frozen student later, and only then feeds mask features into CAD classification.

Status
Pseudo-masks are acceptable as weak supervision and biomarker scaffolding.
Not allowed
Any claim that segmentation is human-validated or ground truth.
Classifier contract
Segmentation stays upstream. Disease labels must not shape masks by default.

The rule is simple.

These masks are operational tools, not final truth. They exist to standardize structure, extract biomarkers, and keep the CAD branch interpretable. If a later paper wants segmentation claims, it needs a manually reviewed validation subset.

Superficial and deep slabs

Vessel masks

Use CLAHE, vesselness filtering, adaptive thresholding, morphology cleanup, then remove tiny disconnected components.

Central avascular region

FAZ masks

Find the central low-flow component, smooth the contour, then reject masks that drift off-center or fall outside plausible size bounds.

Choriocapillaris slab

Flow-deficit masks

Use local thresholding for dark hypoperfusion regions, followed by morphology and area filtering to suppress speckle noise.

Where the student model belongs.

The student is an optional cleanup stage. It learns the pseudo-mask grammar first, then freezes. It is not a second path for disease supervision, and it should not be trained jointly with the classifier unless a separate experiment proves the masks stay stable.

1. Preprocess OCTA

Normalize intensity, apply CLAHE, resize consistently, run image-level quality control.

2. Generate pseudo-masks

Create deterministic vessel, FAZ, and choriocapillaris flow-deficit masks without CAD labels.

3. Train optional student

Use Attention U-Net or U-Net++ style segmentation only to imitate those pseudo-masks.

4. Freeze and measure

Extract density, skeleton density, FAZ geometry, and flow-deficit burden with fold-specific normalization.

5. Fuse for CAD

Feed OCTA image features, F_mask, biomarkers, and clinical data into the classifier.

Why freeze it

Freezing protects the segmentation representation from drifting toward shortcut patterns that only help CAD labels.

What to review before enabling it

  • Montage quality across slabs and devices
  • Failure rate for implausible FAZ or noisy flow maps
  • Whether student masks reduce variance without hallucinating structure

Attention U-Net or U-Net++.

Both remain valid. The default stays conservative: classical masks first, then a lighter student if visual review says the pseudo-labels are stable enough to support it.

Attention U-Net

Lighter, easier to run inside the notebook, and less likely to memorize noisy pseudo-label artifacts.

  • Good default for early experiments
  • Lower overfitting risk on weak labels
  • May miss fine capillary detail

U-Net++

Stronger multi-scale refinement, useful when vessel continuity and FAZ edge quality become the bottleneck.

  • Better thin-structure recovery potential
  • More capacity for OCTA detail
  • Higher risk of learning pseudo-label noise

Default policy

Start without a student. Add one only if montage review, QC counts, and ablations show a real improvement.

  • Compare classical vs student masks directly
  • Freeze segmentation before CAD training
  • Keep joint training as a later experiment
Do not joint-train by default.

Classifier gradients can distort masks into label-specific artifacts. End-to-end segmentation plus classification belongs in an explicit experimental branch with mask-quality monitoring.

What the dataset must return.

The old placeholder mask fields are no longer enough. The notebook output should expose generated masks, mask-derived biomarkers, and provenance so every experiment can trace whether a result came from direct classical processing or from a frozen student stage.

sup_vessel_mask, deep_vessel_mask

Generated vessel pseudo-masks for the superficial and deep slabs.

sup_faz_mask, deep_faz_mask

Generated FAZ pseudo-masks, with center and plausibility checks.

cc_flow_mask

Generated choriocapillaris flow-deficit pseudo-mask.

biomarkers

Mask-derived OCTA measurements appended to the tabular branch.

mask_source

Use classical or student so every downstream run remains auditable.

qc_flags

Optional but useful: record rejected or suspicious masks so weak labels can be filtered later.
