Phase 16 — Benchmarking

Are We Actually Better?

A complex multimodal model is only worth the trouble if it outperforms simpler alternatives. Phase 16 defines and runs a suite of baseline models -- from logistic regression on clinical features to standard CNNs and vision transformers -- so we can rigorously demonstrate that our proposed architecture delivers statistically significant improvements.

The Honesty Check

A model that is 10x more complex but performs the same as logistic regression is not a contribution -- it is over-engineering. Baselines establish the "bar to beat."

Why Do We Need Baselines?

In medical AI research, every new method must answer the question: "Better than what?" Without baselines, a paper reporting 0.85 AUC is meaningless -- maybe logistic regression also achieves 0.85 AUC on this dataset, in which case the deep learning pipeline is unnecessary complexity. Baselines provide context.

Our baseline suite covers three families of models:

  1. Classical ML on clinical features only: logistic regression, random forest, XGBoost. These tell us how far we can get with just the tabular data.
  2. Standard CNNs on OCTA images only: ResNet50, EfficientNet-B0, ConvNeXt-Tiny (single layer). These tell us how well a standard image classifier performs on a single OCTA layer.
  3. Vision Transformers on OCTA images: ViT-S/16. This tells us how a pure-transformer approach compares to our hybrid CNN-transformer design.

Critically, every baseline uses the identical preprocessing, augmentation, cross-validation splits, optimizer family, epoch budget, and early stopping as the full model. This ensures a fair comparison.

For beginners: A "baseline" is a simpler model that you compare against to show your complex model is worth it. Think of it as the "control group" in an experiment. If your experimental treatment (the fancy multimodal model) is not clearly better than the control (a simple logistic regression), you have not demonstrated anything useful.

Classical ML Baselines (Clinical Features Only)

These models receive only the Phase 2 clinical features (age, sex, BMI, smoking, hypertension, diabetes, dyslipidemia). They do not receive OCTA images, generated masks, or biomarkers. They establish the performance ceiling of clinical-data-only diagnosis.

Logistic Regression

The simplest possible classifier. Learns linear decision boundaries between cohorts. Fast, interpretable (you can read the coefficients), and surprisingly effective when clinical features are well-separated. If logistic regression achieves near-perfect AUC, the imaging pipeline adds little clinical value.

Random Forest

An ensemble of decision trees. Captures non-linear interactions between clinical features (e.g., "age x hypertension" might be more predictive than either alone). Handles missing values natively. Usually outperforms logistic regression on tabular data but is harder to interpret.

XGBoost

Gradient-boosted decision trees. Currently the state-of-the-art for tabular/clinical data in many medical benchmarks. Often beats neural networks on small-to-medium tabular datasets. If XGBoost performs close to our full model, it suggests the imaging data is not adding much.

CNN Baselines (OCTA Images Only)

These models receive only OCTA images (one or three channels). No clinical features, no masks. They establish the performance ceiling of image-only diagnosis.

ResNet50 (Sup only)

A 50-layer residual network, the workhorse of medical imaging for years. Takes the Sup layer as a single-channel input. This is a strong standard baseline.

ResNet50 (3-channel Sup+Deep+CC stacked)

Same ResNet50, but with the three OCTA layers stacked as RGB channels. This tests whether simply stacking layers (like color channels) is enough, or whether our per-layer encoder + attention design is actually better. If stacking works as well as our architecture, the cross-layer attention is unnecessary.

EfficientNet-B0 (Sup only)

A more efficient CNN that achieves ResNet-level accuracy with far fewer parameters through neural architecture search. Tests whether a different CNN design philosophy changes results.

ConvNeXt-Tiny (Sup only, single-layer)

Same backbone as our encoders, but used alone as a full classifier. This is the most direct ablation: the ConvNeXt-Tiny baseline differs from our full model only in that it has one encoder, no pretraining, no cross-layer attention, no tabular, and no masks. The comparison isolates the value of everything our model adds on top of a single ConvNeXt-Tiny.

Transformer Baselines (OCTA Images Only)

ViT-S/16 (Sup only)

A Vision Transformer with patch size 16. Instead of convolutions, it splits the image into patches and applies standard transformer attention. This tests whether the attention mechanism is helpful for OCTA analysis even without our cross-layer design, and whether a pure-transformer approach competes with our hybrid CNN-transformer model.

The Full Model (Our Proposed System)

The proposed model includes:

  • Three independent ConvNeXt-Tiny encoders (Sup, Deep, CC) with SimCLR pretraining
  • Phase 4 projection heads reducing 768-d features to 256-d tokens
  • Phase 6 cross-layer attention with layer-identity embeddings
  • Phase 5 tabular encoder with learned mask-token imputation
  • Phase 8 U-Net++ generated masks + mask encoder (Phase 7)
  • Phase 7 multimodal fusion
  • Phase 9 prototype-augmented classification head
  • Phase 10 weighted cross-entropy with label smoothing
  • Phase 11 synchronized augmentation
  • Phase 12 3-stage training schedule (SimCLR -> frozen -> fine-tune)
  • MC-Dropout uncertainty + temperature calibration

The comparison between all baselines and the full model is the core evidence for the paper's central claim: that multimodal fusion of all three retinal layers + clinical data + generated masks outperforms any single-modality approach.

How We Compare (Statistical Rigor)

A simple "our model gets 0.88 AUC, baseline gets 0.85 AUC" is not sufficient. We need to show the difference is statistically significant -- meaning it is unlikely to have occurred by chance.

We use three complementary statistical tests:

TestWhat It ComparesWhen It Matters
McNemar's testBinary correct/incorrect vectors across all test samplesDirect head-to-head: does model A get more samples right than model B?
DeLong testPairwise AUC comparisonStandard in medical ML for comparing ROC curves
Cohen's hEffect size for AUC differencesIs a significant difference also practically meaningful?

We apply Bonferroni correction for multiple comparisons (if we compare against 8 baselines, we adjust the significance threshold to 0.05/8 = 0.00625). All tests use patient-level bootstrap to account for bilateral eye dependence.

Key concept: Statistical significance (p < 0.05) tells you the difference is real, not random. Effect size (Cohen's h) tells you if the difference is large enough to matter clinically. A tiny difference can be statistically significant with enough data, but if AUC improves from 0.850 to 0.853, is that worth the extra complexity?

How Baselines Are Trained

All CNN/transformer baselines follow the same protocol:

  • Same preprocessing pipeline as Phase 2 (CLAHE, resize, z-score)
  • Same Phase 11 augmentations (flips, rotation, affine, mild color jitter, noise)
  • Same 5 fixed Phase 2 folds
  • ImageNet initialization (no SimCLR pretraining for baselines)
  • AdamW optimizer, same LR schedule as Phase 12 Stage 3
  • Same epoch budget and early stopping on validation macro-F1
  • Same evaluation: out-of-fold predictions with 95% bootstrap CIs

Classical ML baselines use the preprocessed clinical features directly, without augmentation. Hyperparameters (regularization strength, tree depth, learning rate) are tuned via inner cross-validation within each training fold.

Glossary for Beginners

Logistic Regression
A linear model that predicts class probabilities. Despite its simplicity, it is still a strong baseline for tabular data and highly interpretable -- you can see which features matter by looking at the coefficients.
Random Forest
An ensemble of many decision trees, each trained on a random subset of data and features. The final prediction is the average (regression) or majority vote (classification) of all trees.
XGBoost
Extreme Gradient Boosting. Builds trees sequentially, where each new tree tries to correct the errors of the previous ensemble. Often the best-performing method on tabular data.
ResNet50
A 50-layer convolutional neural network with residual connections (skip connections that let gradients flow directly through the network). A standard backbone in medical imaging.
ViT-S/16
Vision Transformer Small with 16x16 pixel patches. Treats an image as a sequence of patches and applies transformer encoder layers, similar to how BERT processes text.
Bonferroni Correction
When you run many statistical tests simultaneously, the chance of a false positive increases. Bonferroni divides the significance threshold (0.05) by the number of tests to keep the overall false positive rate at 5%.
