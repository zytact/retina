Phase 11

Data augmentation

Training-only augmentation that preserves vascular structure, keeps masks in spatial alignment with images, and applies mild noise to clinical features. Validation and test data are never augmented.

OCTA image augmentations

Conservative transforms chosen to preserve vessel topology, FAZ morphology, and choriocapillaris texture. The guiding principle: if an augmentation would change clinical interpretation of the scan, it is not applied.

Horizontal flip

p = 0.5

Mirror symmetry is anatomically reasonable. OCTA scans of left and right eyes are already mirrored.

Vertical flip

p = 0.3

Less common than horizontal, but still preserves vessel connectivity.

Rotation

degrees = ±15

Small rotations preserve the circular scan geometry. Larger rotations would crop meaningful peripheral vessels.

Affine shift

translate 5%, scale 0.95–1.05, shear ±5°

Mild spatial distortions. Small enough that FAZ area and vessel density remain stable.

Color jitter

brightness ±0.1, contrast ±0.1

OCTA images are grayscale, but small intensity shifts help the model generalise across acquisition variability.

Gaussian noise

σ = 0.01, p = 0.2

Adds mild sensor-noise simulation. Not applied to mask or clinical branches.

Mask-aware spatial transforms

When OCTA images are geometrically transformed, the corresponding generated masks must receive the identical transform to preserve spatial correspondence.

Same seed, same transform

For each training sample, a single random seed is generated. That seed drives the flip, rotation, affine, and crop parameters. The same parameters are applied to the Sup, Deep, and CC images and to all five generated masks (vessel, FAZ, flow deficit for each layer).

Masks use nearest-neighbour interpolation during geometric transforms to avoid introducing intermediate intensity values. Only the geometric transforms are shared — intensity jitter and Gaussian noise are applied to images only, never to masks.

Clinical feature augmentation

Mild augmentation applied only to the clinical tabular features during training. Generated biomarkers are not augmented.

Continuous feature noise

σ = 0.05, p = 0.3

Small Gaussian noise on z-scored continuous features (age, BMI). Too much noise would corrupt the physiological meaning of these values.

Random feature masking

p = 0.1, one feature per sample

Randomly sets one observed clinical feature to the learned missing-value mask token. This simulates real-world missingness and makes the tabular encoder robust to absent data at inference.

Forbidden augmentations

These transforms are common in natural-image pipelines but would destroy or distort the vascular signal in OCTA. They are explicitly excluded.

Random erasing / Cutout

Removes a rectangular region of the image. In OCTA, this would delete vessel segments, creating false avascular zones that the model could learn as spurious correlations.

Aggressive color jitter

Large brightness or contrast shifts can wash out capillary-level contrast. The vessel density signal depends on fine intensity differences.

Large elastic deformation

Distorts vessel geometry and FAZ morphology. Even if the classification task is disease prediction, not segmentation, the model should learn from real vascular shapes, not warped ones.

Grayscale conversion

OCTA images are already single-channel. Applying random grayscale would be a no-op or could add unintended colour-channel artefacts.

Coarse dropout

Drops entire channels or large feature map regions. In OCTA, this is equivalent to removing an entire vascular layer, which the model could use as a shortcut signal.

Cross-layer synchronisation

All three OCTA layers from the same eye are augmented with the same random seed, preserving the spatial correspondence that the cross-layer attention module relies on.

Why synchronisation matters

The cross-layer attention module in Phase 6 expects that a pixel at position (i, j) in the Sup image corresponds to roughly the same retinal location in the Deep and CC images. If each layer were augmented with an independent random seed, a vessel in the Sup image might be flipped horizontally while the same vessel in the Deep image is not — destroying the layer-to-layer spatial correspondence.

Validation and test data bypass all augmentation. Only the training DataLoader applies these transforms.
