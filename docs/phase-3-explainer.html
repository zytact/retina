Phase 3 of the Retina CAD pipeline

OCTA-SimCLR pretraining before diagnosis training.

Phase 3 teaches each retinal-layer encoder to understand OCTA structure before labels are used for disease classification. Instead of jumping straight from ImageNet into CAD prediction, we first adapt the visual backbone to grayscale vessel anatomy, bilateral similarity, OCTA-specific texture statistics, and the difference between patient-matched versus unrelated vascular patterns.

Core idea

Learn retinal representations first. Classify disease after.

  • Three separate pretraining jobs: SVC, DVC, and CC.
  • OD/OS of the same patient become medically meaningful positive pairs.
  • Single-eye patients still contribute via two augmented views of one image.
  • Projection heads exist only during pretraining, then get discarded.
  • The model learns by comparing embeddings, not by seeing CAD labels yet.

What Phase 3 is.

Phase 3 is a self-supervised learning stage. The model sees OCTA images without disease labels and learns which images should land near each other in feature space. For this project, “near” means either two eyes from the same patient in the same layer or two careful augmentations of the same image when only one eye is available. In short: the supervision comes from image relationships we construct from the data itself, not from CAD labels typed in by a human.

What “self-supervised” means

No one tells the model “this image is CAD” or “this image is control” during Phase 3. Instead, the training pipeline automatically creates learning targets by saying which two views should match and which other images in the batch should not.

What the model actually learns

It learns an embedding space: a numeric representation where vessel maps with shared anatomy and disease context sit closer together, while unrelated retinal patterns sit farther apart.

Why this matters later

When CAD labels are finally introduced in the next phase, the backbone is no longer starting blind. It already knows common OCTA structures, layer-specific textures, and how to encode meaningful vascular similarity.

Input

Preprocessed grayscale OCTA images, separated by retinal layer: superficial, deep, and choriocapillaris.

Objective

Make medically related images embed close together and unrelated images push apart.

Output

Three OCTA-adapted ConvNeXt-Tiny backbones, ready for downstream multimodal fine-tuning.

What self-supervised learning is.

Self-supervised learning is a way to train a model without manual labels by generating a training signal from the raw data itself. In SimCLR, the question is not “which disease class is this?” The question is “which two images should represent the same underlying case, and which images should represent different cases?”

Supervised learning

Needs explicit labels such as CAD versus non-CAD. The model directly maps an image to a diagnosis target.

Self-supervised learning

Uses automatically constructed targets. Here, the target is relational: matched OCTA views should agree in embedding space even before any diagnosis label is used.

Why it fits OCTA

OCTA collections often contain more useful images than high-confidence labels. Self-supervision lets the project extract signal from every scan by first learning retinal structure, then learning disease classification.

Is this clustering?

It is similar to clustering in the sense that similar samples are encouraged to live near each other, but it is not the same thing. Phase 3 does not explicitly create disease groups or assign cluster IDs.

What it does instead

It learns a useful embedding space — a map of OCTA similarity. In that map, related eyes and related image views move closer, while unrelated cases move farther apart.

What comes later

Once that map is learned, later stages add self-generated OCTA masks/biomarkers and then use real labels such as control versus disease or specific clinical cohorts to place decision boundaries on top of the embedding.

Think of Phase 3 as pretraining the model to speak the visual language of OCTA. The later classifier then uses that language for CAD prediction. A good shorthand is: clustering tries to make groups, self-supervised contrastive learning tries to learn a good map.

Why we are doing it.

Skipping this stage would mean asking a backbone trained on natural RGB photographs to immediately understand grayscale vascular maps. That is risky. Phase 3 reduces the domain gap, uses unlabeled data efficiently, and gives later classification stages a stronger anatomical prior.

1. Domain gap

ImageNet teaches edges, objects, fur, sky, texture, and color relationships. OCTA needs vessel continuity, capillary density, flow voids, and layer-specific vascular geometry.

2. Label efficiency

Self-supervised training lets every OCTA image help shape the backbone before class labels are consumed in the harder downstream task.

3. Better fine-tuning start

The classifier begins from features already aligned to retinal anatomy, so later disease learning is more stable and less data-hungry.

Why bilateral pairs matter.

The plan uses a medically meaningful contrastive signal unique to ophthalmic data. When a patient has both eyes available, OD and OS in the same OCTA layer share systemic disease context. They are not identical images, but they are biologically related views of the same disease state.

Visualization 2 — positive pair strategy

The upper path uses bilateral natural positives. The lower path is the fallback for unilateral cases. Both routes feed the same contrastive objective.

Why this is strong: it preserves disease consistency while still allowing natural anatomical variation. That teaches the encoder which changes are disease-relevant versus incidental.

How the learning works.

Each image goes through a layer-specific ConvNeXt-Tiny backbone and a temporary projection head. SimCLR then compares normalized embeddings with NT-Xent loss. Positive pairs are pulled together; the rest of the batch acts as negatives. After pretraining, the projection head is removed and only backbone weights are kept. The key learning signal is simple: if two embeddings came from the same underlying case, similarity should increase; if they came from unrelated cases, similarity should decrease.

Visualization 3 — Phase 3 mechanics

One layer at a time: augment, encode, project, compare, optimize, discard head, keep adapted backbone.

Important detail: the three retinal layers do not share weights during pretraining. Each layer gets its own encoder because superficial, deep, and CC textures are biologically different.

Step 1 — turn image into features

The ConvNeXt backbone converts each OCTA image into a compact feature vector. Early layers respond to local vessel edges and contrast transitions; deeper layers summarize larger vascular arrangements and texture patterns.

Step 2 — compare paired embeddings

The projection head maps features into a contrastive space where cosine similarity is measured. If the pair is OD/OS from one patient, or two augmentations of one eye, the loss rewards those embeddings for becoming closer.

Step 3 — separate unrelated cases

All other images in the batch become implicit negatives. Their embeddings compete with the true match, so the backbone must keep patient-matched vascular patterns aligned while not collapsing every OCTA scan into the same representation.

Conservative augmentations

Horizontal flip, mild rotation, modest crop-resize, Gaussian blur, and light intensity jitter. These perturb the image without destroying vessel structure.

What we avoid

No aggressive distortions, no random erasing, no inversion tricks. Those may improve generic SSL benchmarks but can erase clinically meaningful vascular signal.

Why projection heads are temporary

The projection head helps the contrastive objective work well, but the downstream classifier benefits most from the backbone representation, not the pretraining-only head.

A useful mental model: every training batch behaves like a small ranking problem. For each anchor OCTA image, the model should rank its true partner as most similar and rank unrelated patients lower.

Training schedule and expected payoff.

Phase 3 is not the final model. It is the preparation stage that gives later image encoders better starting weights. The updated plan then adds pseudo-mask generation, optional segmentation-student refinement, multimodal fusion, and classification. The schedule stays modular so each layer and each generated mask family can be audited independently.
