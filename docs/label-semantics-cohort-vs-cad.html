Label semantics for Version 1

The model is learning RASTA cohort identity, not a direct binary CAD endpoint.

The folder tree, notebook artifacts, and publication table all point to the same operational target: a four-class classifier over RASTA clinical/study cohorts. CAD is present in the clinical context, especially in MRCC and vascular disease fields, but it is not exposed as a clean universal yes/no label in the current files.

Current supervised target: RETINORM, ORNET, FAMILIPO, or MRCC.

This is a legitimate and reproducible Version 1 target because it follows the dataset organization. It should be framed as cohort classification for cardiovascular-risk-associated OCTA populations, not as standalone CAD diagnosis.

What the four folders mean

The RASTA directory is organized into four cohort folders in this local project copy. These are not four arbitrary labels. They correspond to control, sleep apnea, familial hypercholesterolemia, and coronary revascularization populations described in the RASTA data descriptor.

CAD-risk adjacent. Familial hypercholesterolemia is strongly linked to atherosclerotic risk, but the folder itself is a cohort label.

Most CAD-related. This is the clearest coronary disease population because it involves coronary revascularization surgery.

The Excel file is metadata, not the class system

Clinical covariates

Age, sex, body mass index, hypertension, diabetes mellitus, stroke, smoking, dyslipidemia, sleep apnea, and heart failure describe patient status.

Risk scoring

CHA2DS2-VASc appears as a cardiovascular risk score. It can support analysis or future endpoint design, but it is not the current class label.

Vascular disease

The field captures broader vascular disease, including myocardial infarction among other vascular conditions. It is broader than a CAD-only endpoint.

OCTA biomarkers

FAZ, vessel density, and perfusion density columns are quantitative retinal features that can be inputs or comparison variables.

How to read the table

Diabetes and hypertension are not separate output classes here. They are covariates that may explain or confound cardiovascular risk, and they may improve prediction when fused with OCTA imaging.

The class label still comes from the folder cohort.

What this means for the plan

The current plan is technically consistent with four-class cohort classification. The main correction is framing: avoid claiming direct CAD diagnosis unless a future phase defines and validates a CAD endpoint.

Safe Version 1 claim

InputSup, Deep, CC OCTA layers plus clinical metadata and generated biomarkers
TargetRASTA folder-derived cohort label
ClaimMultimodal classification of cardiovascular-risk-associated retinal OCTA cohorts

Future CAD direction

A binary CAD model would need a separate endpoint definition. Candidate sources could include MRCC membership, the vascular disease field, coronary calcium context, or a clinically reviewed CAD label. These are not interchangeable without clinical validation.

Recommended wording

“Version 1 predicts RASTA cohort membership across RETINORM, ORNET, FAMILIPO, and MRCC. CAD-related status is represented indirectly, especially through the MRCC cohort and vascular disease covariates. Direct binary CAD prediction is a future endpoint-definition task.”
