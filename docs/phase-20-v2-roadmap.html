What Comes Next?

Version 1 lays the foundation: 2D OCTA en-face images processed through layer-specific encoders with cross-layer attention, clinical fusion, and generated mask guidance. Version 2 extends this to 3D angiocubes -- the full volumetric OCTA data -- with a 3D backbone, self-supervised masked reconstruction, and multi-task learning.

V1 to V2

V1 is explicitly designed to be V2-compatible. The modular architecture (per-layer encoders -> cross-layer attention -> multimodal fusion -> classification head) can accept 3D encoder outputs without changing the downstream modules.

Why V2? What is Missing in V1?

V1 uses 2D en-face OCTA images -- the projection of each retinal layer onto a flat plane. This loses information: the depth distribution of vessels within each layer, the 3D morphology of capillary networks, and volumetric flow patterns. OCTA scanners actually capture 3D angiocubes (x, y, depth) with rich depth-resolved information that the en-face projection discards.

V2 aims to recover this lost information by processing the full 3D angiocube directly. This promises:

  • Richer biomarkers: volumetric vessel density, 3D FAZ morphology, depth-resolved flow deficits
  • Better layer separation: instead of relying on the scanner's固定的 slab segmentation, the model can learn optimal depth partitions
  • Stronger cross-layer interaction: depth-wise attention can model how SVC vessels feed into DVC capillaries in 3D
For beginners: Think of it like comparing a photograph (2D) to a CT scan (3D). A photograph shows you the surface. A CT scan shows you the internal structure. V1 uses photographs of each retinal layer. V2 will use the full 3D scan.

What Are Angiocubes?

An OCTA angiocube is a 3D volumetric scan of the retina, with dimensions approximately [304, 304, 304] voxels (x, y, depth), where each voxel represents the blood flow signal at that 3D location. The depth direction spans from the inner limiting membrane to the choroid, approximately 2-3 mm of tissue.

The scanner's software traditionally segments this volume into slabs corresponding to the SVC, DVC, and CC layers, then projects each slab to produce the en-face images V1 uses. V2 will skip this projection and learn directly from the raw or minimally-processed volume.

Replacing 2D ConvNeXt-Tiny with 3D Swin-T

The backbone upgrade is the core architectural change in V2:

ComponentV1 (2D)V2 (3D)
BackboneConvNeXt-Tiny (2D conv)3D Swin-T (3D shifted-window transformer)
Input3 x [1, 224, 224] grayscale images1 x [D, 224, 224] angiocube (depth D ~304)
Encoders3 independent per-layer encoders1 shared backbone + layer-specific adapter tokens
Layer infoLayer-identity embeddings in attentionLearned adapter tokens per depth range
Patch size4x42x2x2 (spatiotemporal patches)

The 3D Swin-T processes the angiocube volume directly, outputting a 3D feature map. Layer-specific information is extracted via adapter tokens: learnable query vectors that attend to specific depth ranges in the Swin-T output, producing per-layer feature representations that can feed into the existing cross-layer attention module without modification.

Retinal-MAE: Self-Supervised Pretraining for Angiocubes

V1 uses SimCLR (contrastive learning) with bilateral positives. V2 can use a more powerful self-supervised method: Masked Autoencoding (MAE) adapted for retinal angiocubes.

In Retinal-MAE:

  • Randomly mask 75% of 3D patches in the angiocube (using cube-shaped masks to preserve spatial structure)
  • Encode only the visible patches through the 3D Swin-T encoder
  • Decode the full volume with a lightweight 3D decoder
  • Loss: MSE between reconstructed and original voxel intensities in the masked region only

The layer-masked variant adds an additional pretraining signal: mask entire depth slabs corresponding to anatomical layers, forcing the model to infer one layer's appearance from the other layers. This teaches cross-layer anatomical knowledge that transfers directly to the downstream classification task.

Multi-Task Learning with PCGrad

V2 adds multi-task learning, simultaneously optimizing three objectives:

  • Disease classification: The primary task (same as V1)
  • Biomarker regression: Predicting quantitative OCTA metrics (vessel density, FAZ area, etc.) directly from the 3D volume, supervised by the device-extracted biomarkers
  • Volume reconstruction: A decoder branch reconstructs the input angiocube as an auxiliary task, providing a strong regularization signal

Multi-task learning with shared representations can improve generalization, but it introduces a problem: the three loss gradients may conflict (one task's optimal update harms another task). V2 addresses this with PCGrad (Project Conflicting Gradients), which projects conflicting gradient components onto the normal plane of the other task's gradient, preventing destructive interference.

V1 to V2 Compatibility

V1's architecture was designed with V2 in mind. The key insight: the cross-layer attention module (Phase 6), multimodal fusion (Phase 7), and classification head (Phase 9) are agnostic to whether the per-layer tokens come from 2D encoders or 3D adapter tokens. The token dimension is 256 in both cases.

This means:

  • V2 can reuse V1's Phase 5 (tabular encoder), Phase 7 (fusion), Phase 9 (prototype classifier), Phase 10 (losses), Phase 11 (augmentation), Phase 14 (evaluation), and Phase 15-18 (ablation/baselines/stats/contributions) with minimal or no changes.
  • The main V2 work is: new 3D backbone (3D Swin-T), new pretraining (Retinal-MAE), new adapter tokens, new multi-task head (biomarker regression + reconstruction decoder), and PCGrad loss balancing.
  • The V1 ablation results serve as the baseline for V2: the question becomes "does adding 3D information improve over the best 2D multimodal system?" rather than "does any deep learning work at all?"

Glossary for Beginners

Angiocube
A 3D volumetric OCTA scan. Instead of flattening the retina into layers (2D), it captures blood flow signals at every 3D coordinate within the scanned volume. Think of it as a 3D movie of blood flow through the retinal microvasculature.
3D Swin-T
A 3D version of the Swin Transformer. It processes volumetric data by applying self-attention within shifted 3D windows, capturing both spatial and depth-wise relationships. The "T" stands for Tiny (smallest variant).
Masked Autoencoding (MAE)
A self-supervised learning method where a large fraction of input patches (usually 75%) are masked, and the model must reconstruct the original data from the visible patches. This forces the model to learn holistic representations of the data structure.
Adapter Token
A learnable query vector that extracts information about a specific aspect of the data from a shared backbone. In V2, adapter tokens for SVC, DVC, and CC attend to their respective depth ranges in the 3D Swin-T output, producing per-layer features without needing separate encoders.
PCGrad
Project Conflicting Gradients. A method for multi-task learning that detects when gradients from different tasks point in conflicting directions and projects them to remove the conflicting component. This prevents one task from undoing another task's learning progress.
ETDRS Grid
Early Treatment Diabetic Retinopathy Study grid. A standardized partitioning of the macula into concentric rings (center, inner, outer) and quadrants (superior, inferior, nasal, temporal) for localized analysis. In V2, biomarkers are computed per ETDRS zone within the 3D volume.
