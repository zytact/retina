Why Explainability Matters

Medical AI cannot be a black box. Regulators, clinicians, and patients all need to understand—at minimum—what drove a model's decision.

Deep learning models for medical image analysis are often described as "black boxes": they map pixels to predictions through millions of learned parameters in a way that is opaque to human inspection. For a retinal screening tool that predicts coronary artery disease risk from OCTA images, this opacity is not acceptable. A clinician who sees a "high risk" prediction needs to know: Which part of the retina drove that decision? Was it the superficial capillary layer, the deep capillary layer, or the choriocapillaris? Which clinical risk factors contributed most?

Explainability in medical AI serves several distinct purposes:

  • Clinical trust — Clinicians are far more likely to trust and adopt a system whose reasoning they can examine and verify against their own expertise.
  • Diagnostic insight — The explanations themselves may reveal novel imaging biomarkers. If the model consistently focuses on a specific vascular region in patients with familial hypercholesterolemia, that region may warrant further clinical investigation.
  • Bias detection — Heatmaps can reveal shortcut learning. A model that focuses on the image border, a watermark, or a patient ID label is not learning clinically relevant features—it is cheating. Explainability exposes this.
  • Regulatory compliance — Frameworks such as the EU AI Act and FDA guidance for AI/ML-enabled medical devices increasingly require transparent and interpretable model behavior.
  • Error analysis — When the model is wrong, explainability helps answer why. Was it distracted by an artifact? Did it overweigh a particular clinical feature? Understanding failure modes guides targeted improvements.
Three complementary views. This phase implements three distinct explanation modalities—spatial (where in the image), attentional (which retinal layer), and feature-based (which clinical variables)—and then integrates them into a unified diagnostic narrative for each patient.

No single explanation method is perfect. Each has blind spots. By combining Grad-CAM (spatial attribution), attention-dominance analysis (layer attribution), and SHAP (feature attribution), we triangulate on the model's reasoning from three independent angles. When they agree, we can be more confident. When they disagree, that tension itself is informative.

The sections that follow explain each method in detail: what it is, how it works mathematically, what it produces visually, how to interpret the outputs, and what they mean specifically for the Retina CAD project.

Grad-CAM — Spatial Heatmaps Per Layer

Gradient-weighted Class Activation Mapping answers the most basic question: which pixels mattered most?

What is Grad-CAM?

Grad-CAM (Gradient-weighted Class Activation Mapping) is a technique that produces a coarse localization heatmap highlighting the regions of an input image that were most influential for a particular class prediction. It was introduced by Selvaraju et al. (2017) and has become a standard explainability tool in computer vision, especially for medical imaging.

Unlike some older methods (like occlusion sensitivity, which requires many forward passes), Grad-CAM requires only a single forward and backward pass through the network, making it computationally efficient enough to run on every test sample.

How it works: the intuition

Deep convolutional networks learn increasingly abstract features as information flows through their layers. Early layers detect edges and texture; middle layers detect shapes and patterns; the final convolutional layer before global average pooling contains the highest-level spatial feature maps—each channel activates in response to a particular visual concept (e.g., "vessel-like structures," "dark regions," or "textured areas").

Grad-CAM works by asking: "Which of these high-level feature maps are most important for predicting class X?" It measures importance by computing the gradient of the class score with respect to each feature map. A feature map that has a strong positive gradient for class X is one that the model considers evidence for class X. The heatmap is a weighted combination of these feature maps, showing where the model looked to make its decision.

Grad-CAM computation

For a given class c and feature map Ak at the final convolutional layer:

αkc = 1/Z · ∑i ∑j ∂yc / ∂Akij

LcGrad-CAM = ReLU( ∑k αkc · Ak )

αkc is the importance weight for feature map k with respect to class c, computed by global-average-pooling the gradients. The final heatmap is a weighted sum of feature maps, passed through ReLU to show only positive (evidence-for) regions. The heatmap is then upsampled to the input image resolution.

Why the ReLU?

Grad-CAM applies ReLU after summing weighted feature maps. This zeroes out negative contributions— pixels whose presence would push the model away from class c. The result is a heatmap that shows only regions that provide positive evidence for the target class. This is clinically appropriate: we want to see what the model uses to diagnose a condition, not what it uses to rule it out (though that can also be informative, and the raw un-ReLU'd map is available for inspection).

Why Grad-CAM for Retina CAD?

In the Retina CAD project, each OCTA scan has three layers: Superficial Vascular Complex (SVC), Deep Vascular Complex (DVC), and Choriocapillaris (CC). These layers are processed by separate encoders because they have fundamentally different vascular architecture. Grad-CAM is applied independently to each encoder's final convolutional feature map.

This produces three distinct heatmaps per patient, per prediction:

  • SVC heatmap — Shows which regions of the superficial vascular plexus drove the classification. In diseases affecting arteriolar density or capillary dropout (e.g., hypertensive retinopathy, diabetic microangiopathy), the SVC heatmap typically highlights regions with abnormal vessel morphology.
  • DVC heatmap — Highlights regions in the deep capillary layer. DVC pathology is often associated with deeper microvascular changes. The DVC is particularly sensitive to diabetic retinopathy and venous occlusive disease.
  • CC heatmap — Shows regions of choriocapillaris perfusion that influenced the decision. CC flow deficits are associated with choroidal ischemia and may correlate with systemic cardiovascular risk.
Clinical intuition: In a patient with familial hypercholesterolemia (FAMILIPO cohort), you might expect the SVC heatmap to highlight peripapillary or macular arterioles (where atherosclerosis manifests), while the CC heatmap might highlight patchy perfusion deficits. In a control patient (RETINORM), the heatmaps would ideally be diffuse or focus on anatomically normal features—or the model might rely more on clinical features, producing weaker spatial attention.

Class-targeted vs. predicted-class heatmaps

The system supports two modes:

  • Predicted-class mode: Grad-CAM is computed for the class the model actually predicted. This answers: "Why did you choose this diagnosis?"
  • Class-targeted mode: Grad-CAM is computed for a specific class of interest (e.g., MRCC), regardless of what the model predicted. This answers: "What evidence would you need to see to diagnose this condition in this patient?"

Both modes are diagnostically useful and are generated during the Phase 13 explainability run.

Cross-cohort heatmap comparison

One of the most informative analyses is comparing typical heatmap patterns across cohorts. For each cohort (RETINORM, ORNET, FAMILIPO, MRCC), the system aggregates heatmaps from all correctly classified patients to identify cohort-specific spatial patterns.

Expected patterns by cohort:

  • RETINORM (Control) — Diffuse, low-intensity heatmaps across all layers. The model may rely more on clinical features ("no cardiovascular risk factors") than on spatial features, producing weaker Grad-CAM signals.
  • ORNET (Sleep Apnea) — Possible SVC and DVC hotspots in the macular region, consistent with known effects of intermittent hypoxia on retinal microvasculature.
  • FAMILIPO (Hypercholesterolemia) — SVC heatmaps may highlight arteriolar regions. CC heatmaps may show peripapillary flow-deficit patterns reflecting choroidal atherosclerosis.
  • MRCC (Cardiac surgery / CAD) — The most severe cardiovascular cohort. Likely multi-layer involvement: SVC capillary dropout, DVC rarefaction, and CC flow deficits all contributing to the prediction.
Why this matters. If the expected cohort-specific patterns match clinical knowledge, it validates that the model learned biologically plausible features. If they do not—if the model focuses on image artifacts, watermarks, or unexpected regions—it signals shortcut learning that must be addressed before deployment.

Attention Rollout — Layer Dominance Analysis

Which retinal layer does the model rely on most? The cross-layer attention module reveals each layer's contribution to every prediction.

The cross-layer attention mechanism

Before explainability, recall how the model processes retinal layers. Phase 6 of the pipeline implements a cross-layer attention module that operates on the three encoded layer tokens:

SVC token → Layer-identity embedding → DVC token → Layer-identity embedding → CC token → Layer-identity embedding → Multi-head Self-Attention → Mean Pool → Focta

The three tokens (each 256-dimensional) are stacked into a sequence S = [TSup; TDeep; TCC] and pass through a transformer encoder block with 8 attention heads. The attention weight matrix A ∈ ℝ3×3 captures how much each layer "attends to" every other layer.

Multi-head self-attention on layer tokens

Given token sequence S ∈ ℝ3 × 256:

Attention(Q, K, V) = softmax(QKT / √256) · V

Headh = Attention(SWhQ, SWhK, SWhV)

MultiHead(S) = Concat(Head1, ..., Head8) · WO

Each head produces a 3×3 attention matrix, where entry (i, j) represents how much layer i "attends to" layer j. The matrices are averaged across heads to produce a single per-sample 3×3 attention matrix.

Layer-dominance scores

From the per-sample attention matrix A ∈ ℝ3×3, we compute a layer-dominance score for each retinal layer:

Layer-dominance score

dominancek = (1/Nqueries) · ∑i Aik

for k ∈ {Sup, Deep, CC}

The dominance score for layer k is the average attention that all query tokens (rows) pay to layer k (column k). A higher score means that, across all three tokens attending to each other, layer k's features received the most "attention weight."

This produces three scores per sample that sum to 1.0. A dominance of (0.5, 0.3, 0.2) means the SVC contributed roughly half of the layer-attention signal, while CC contributed the least.

Clinical interpretation of dominance scores

Layer dominance tells us which retinal layer's encoded features the model found most informative for classifying a particular patient. When aggregated across a cohort, it tests hypotheses about disease pathophysiology:

Why this matters clinically:

  • SVC dominance suggests the model identifies disease primarily through superficial vascular changes—arteriolar narrowing, capillary dropout, or tortuosity visible in the superficial plexus.
  • Deep dominance suggests deeper capillary changes, often associated with venous insufficiency or diabetic microangiopathy.
  • CC dominance highlights choroidal perfusion. In cardiovascular disease, choroidal flow deficits may parallel systemic endothelial dysfunction, making the CC a "window" into systemic vascular health.

Patient-level vs. cohort-level analysis

The system computes dominance at both levels:

  • Per-patient: Each patient receives (dominanceSup, dominanceDeep, dominanceCC) for their prediction. This enables individual case review—"Why did this specific patient get flagged as high risk? The model focused on the CC."
  • Cohort-level: Dominance scores are averaged across all correctly classified patients in each cohort, with confidence intervals. This tests whether the model consistently relies on the same layers for each disease group.
Important caveat. Attention dominance is a proxy, not proof. It measures what the model attends to, not necessarily what is causally important. Layer-ablation experiments (Phase 15: "Sup only," "Deep only," "CC only") provide stronger causal evidence. Dominance analysis is best used to generate hypotheses that can be tested with ablations.

The attention analysis outputs include:

  • Per-sample attention matrices — Saved as .npy arrays for re-analysis.
  • Cohort dominance summary — CSV with per-cohort mean, std, and 95% CI for each layer.
  • Patient-level dominance CSV — Each row is a patient with their Sup, Deep, CC dominance scores and predicted/true labels.
  • Dominance-visualization plots — Stacked bar charts or ternary diagrams showing the distribution of layer attention across cohorts.

Sample-level analysis: supra-threshold attention

Beyond the mean dominance score, the system also flags patients where a single layer's dominance exceeds a threshold (e.g., > 0.5), indicating the model heavily relied on that layer alone. These "single-layer-reliance" cases are clinically interesting: they may represent patients whose disease has a particularly strong signature in one vascular compartment.

SHAP — Clinical Feature Attribution

Which clinical risk factors drive each prediction? SHAP values attribute the model's output to individual input features, grounded in cooperative game theory.

What is SHAP?

SHAP (SHapley Additive exPlanations) is a unified framework for interpreting model predictions based on Shapley values from cooperative game theory (Lundberg & Lee, 2017). In this framework, each input feature is a "player" in a coalition game, and the "payout" is the model's prediction. The Shapley value for a feature tells us how much that feature contributed to the difference between the actual prediction and the average prediction.

Shapley value for feature i

φi = ∑S ⊆ F \ {i} |S|! (|F| - |S| - 1)! / |F|! · [fS ∪ {i}(xS ∪ {i}) - fS(xS)]

Where F is the set of all features, S is a subset of features not containing i, and fS is the model's prediction conditioned on the features in S. The Shapley value is the average marginal contribution of feature i across all possible feature subsets. For efficiency, the system uses KernelSHAP, an approximation that samples feature subsets rather than enumerating all 2|F| combinations.

The key property that makes SHAP valuable: Shapley values are the only attribution method that satisfies four desirable axioms—efficiency (the sum of all Shapley values plus the baseline equals the prediction), symmetry (equal-contribution features get equal values), dummy (a feature that never changes the prediction gets zero attribution), and additivity (the attributions of a sum of models equal the sum of individual model attributions).

What features does SHAP analyze?

The tabular encoder in the Retina CAD model takes two groups of features:

Clinical features

Age, sex, BMI, smoking status (never/former/current), hypertension, diabetes, dyslipidemia, plus additional cardiovascular covariates from the RASTA spreadsheet (stroke, vascular disease, CHA2DS2-VASc components).

Generated OCTA biomarkers

Vessel density (Sup, Deep), vessel skeleton density, FAZ area/perimeter/circularity/centroid distance (Sup, Deep), CC flow-deficit percentage/count/mean area/median area.

SHAP attributions are computed separately for clinical features and generated biomarkers. This prevents the larger number of biomarker features from diluting the clinical signal and allows independent interpretation of each modality.

Visualizations produced

The SHAP analysis generates several types of plots, each serving a different analytical purpose:

Per-patient waterfall plots

A waterfall plot shows the prediction for a single patient as an additive decomposition. It starts at the model's expected (average) output (E[f(x)]), then shows each feature pushing the prediction up or down by its Shapley value. The final value is the model's actual prediction for that patient.

E[f(x)] = 0.25 (baseline)

- Smoking = 0 → -0.03

In this hypothetical example, advanced age, hypertension, and elevated Sup vessel density and CC flow deficit all pushed the prediction upward. Note that SHAP shows direction (positive = pushes toward the cohort, negative = pushes away).

Population beeswarm plots

A beeswarm (or summary) plot shows the SHAP values for every feature across all patients. Each dot represents one patient. The x-axis is the SHAP value (impact on model output), feature values are color-coded (red = high, blue = low), and features are ranked by their overall importance (mean absolute SHAP value).

Reading a beeswarm plot
  • Top features are the most influential overall (largest mean |SHAP|).
  • Spread indicates variability: a wide spread means the feature sometimes pushes predictions up, sometimes down.
  • Color gradient shows direction: if high feature values (red) cluster on the positive side, the feature has a monotonic positive relationship with the outcome.
  • Gaps or clusters may reveal nonlinear relationships or threshold effects (e.g., age > 65 suddenly increases CAD risk).

What we expect to learn:

  • Which clinical features are most discriminative for each cohort.
  • Whether generated biomarkers (vessel density, FAZ area, CC flow deficit) add independent predictive value beyond traditional risk factors.
  • Whether the model learned expected relationships (e.g., older age, hypertension, and diabetes should increase predicted risk for MRCC) or unexpected ones (a red flag for data or labeling issues).
  • Whether there are features with near-zero SHAP values across all patients—candidates for removal in a simplified model.

Force plots

Force plots are an alternative visualization that "pushes" the prediction from the base value. They are particularly useful for clinical case review: a single force plot communicates both the direction and magnitude of each feature's contribution at a glance.

The system generates force plots for representative patients from each cohort, selected as the patients whose SHAP profiles are closest to the cohort's mean SHAP profile (the "typical" case) and those farthest from it (the "atypical" case).

SHAP for generated biomarkers: validating the mask pipeline

One particularly important use of SHAP is validating that the self-generated segmentation masks (Phase 8) produce clinically meaningful biomarkers. If vessel density, FAZ area, and CC flow deficit have high SHAP values, it confirms that the automated mask pipeline captures informative features. If they have near-zero SHAP values despite the model achieving good classification performance, it suggests the model relies on image features that are not captured by the summary biomarkers— which is itself an informative finding.

Integrated Analysis

The three explanation modalities are strongest when combined. The integrated analysis correlates spatial heatmaps, layer dominance, and feature attributions into a cohesive diagnostic narrative.

Cross-modal consistency

Each explanation method has blind spots. Grad-CAM sees spatial regions but not clinical features. SHAP sees clinical features but not spatial locations. Attention dominance sees which layer matters but not where in that layer. The integrated analysis brings them together.

Grad-CAM ↔ Attention dominance

If the attention-dominance analysis says "SVC is the dominant layer for FAMILIPO patients," the integrated analysis checks: do the SVC Grad-CAM heatmaps for FAMILIPO patients show stronger, more focused activation than the Deep or CC heatmaps? If both methods agree, it is strong evidence that the model genuinely relies on SVC features. If they disagree— if attention says SVC is dominant but the CC heatmap is the only one with meaningful structure—it deserves investigation.

Correlation metrics produced:
  • Per-layer Grad-CAM mean intensity vs. per-layer dominance score (Pearson r)
  • Fraction of patients where the layer with the highest Grad-CAM intensity matches the layer with the highest dominance score
  • Per-cohort consistency matrices

Grad-CAM ↔ SHAP

This is the most clinically informative cross-reference. For a patient where SHAP indicates that "CC flow deficit" is a strong positive contributor, the integrated analysis checks the CC Grad-CAM heatmap: does the spatial pattern of activation correspond to regions of flow deficit visible in the CC scan? If so, the model is using the same pathology the clinician would see. If the SHAP signal is strong but the heatmap shows diffuse or anatomically implausible activation, the model may be relying on spurious correlations.

Attention dominance ↔ SHAP

If the SVC layer dominates attention, do SVC-derived biomarkers (Sup vessel density, Sup FAZ area) also have high SHAP values? A consistent picture across both modalities strengthens the case that superficial vascular changes are genuinely driving the classification.

Cohort-level integrated fingerprints

At the cohort level, the integrated analysis produces "explainability fingerprints"—multivariate profiles that combine layer dominance, Grad-CAM spatial metrics (e.g., fraction of heatmap mass within the central 30° field), and top SHAP features. These fingerprints can reveal whether each cohort has a distinct explainability signature, which would validate both the model and the biological hypothesis underlying the project.

Scientific value. Consistent explainability fingerprints across cohorts would provide evidence that the model has learned biologically meaningful and cohort-specific features, not dataset-level shortcuts. Inconsistent or overlapping fingerprints would warrant investigation into whether the four cohort labels are actually separable in the OCTA domain.

Technical Implementation

How the explainability pipeline is implemented in the notebook: from trained checkpoints to publication-ready visualizations.

Overview of the pipeline

Load fold model → Load temperature → Grad-CAM → Attention → SHAP → Integrated

Step 1: Model loading

Phase 13 loads the trained Phase 12 fold models from checkpoint files. Each fold has its own model (5 folds total) with the corresponding temperature-scaling parameter from post-hoc calibration. Models are set to evaluation mode with the frozen U-Net++ segmentation student also in eval mode.

Step 2: Grad-CAM generation

For each patient in the held-out test set of each fold:

  1. Forward pass through the Sup encoder: capture the final spatial feature map (7×7×768) before global average pooling.
  2. Compute gradients of the predicted-class score (or a specified target class) with respect to the feature map.
  3. Global-average-pool the gradients to get per-channel importance weights.
  4. Compute the weighted sum of feature maps, apply ReLU, upsample to 224×224.
  5. Overlay on the original OCTA image with a colormap (viridis or hot).
  6. Repeat for Deep and CC encoders.
  7. Save the overlay PNG and the raw .npy heatmap.

The system generates heatmaps for correctly classified and misclassified patients separately, enabling comparison and error analysis.

Step 3: Attention extraction

During the forward pass through the Phase 6 cross-layer attention module, the per-sample attention matrix A ∈ ℝ3×3 is extracted (averaged across 8 heads). The layer-dominance scores are computed as described above. Results are aggregated per cohort and saved.

Step 4: SHAP computation

SHAP is applied to the trained tabular encoder using KernelSHAP (from the shap Python library). Because SHAP can be computationally expensive for large feature sets, the system:

  • Computes SHAP values for a representative subset of patients per cohort (e.g., 50 per cohort) rather than all patients.
  • If shap is not installed in the runtime environment, falls back to occlusion-based feature attribution, which perturbs each feature individually and measures the change in model output.
  • Saves per-patient SHAP values, global feature importance (mean |SHAP|), and the visualization plots (beeswarm, waterfall, force).

Step 5: Integrated analysis

The outputs from Steps 2–4 are loaded and cross-referenced:

  • For each patient, the system checks whether the layer with the strongest Grad-CAM signal matches the layer with the highest dominance score.
  • Cross-cohort consistency matrices are computed.
  • An integrated summary CSV is produced with all explanation metrics per patient, and cohort-level fingerprints are written.

Clinical Interpretation Guide

A practical guide for clinicians reviewing the explainability outputs: what to look for, how to distinguish genuine biological signal from noise, and when to flag concerns.

Evaluating a Grad-CAM heatmap

Is the heatmap focused on anatomically plausible regions?

A valid heatmap should highlight regions where OCTA shows vascular structures: the macular region, peripapillary area, and visible vessel arcs. Heatmaps that fire on the image border, on uniform background regions, or on known artifacts (watermarks, scan circles, patient IDs) are red flags.

Does the layer with the strongest activation match clinical expectations?

SVC-dominant diseases (e.g., hypertensive retinopathy, arteriolar changes) should show SVC heatmaps with strong focal activation. CC-dominant diseases (e.g., choroidal ischemia) should show CC heatmaps with patchy flow-deficit patterns. If the model consistently relies on an unexpected layer, it may be learning a confound.

Are incorrect predictions explainable?

When the model misclassifies a patient, the heatmap should ideally show why: perhaps the OCTA image has an unusual artifact, or the patient has atypical anatomy. A heatmap that looks reasonable despite a wrong prediction suggests the model's error is not spatial but may be related to clinical features.

Evaluating layer dominance

  • Cohort consistency: Are dominance patterns consistent within a cohort (low variance) or widely variable? High variance within a cohort suggests the model is not learning a consistent layer-specific signature.
  • Cohort separation: Do different cohorts have measurably different dominance profiles? If all four cohorts show essentially the same pattern (e.g., all SVC-dominant), the layer-dominance analysis adds no discriminative insight.
  • Clinical plausibility: Does the dominance pattern match known pathophysiology? The hypothesis that MRCC patients show elevated CC dominance should be consistent with clinical knowledge about choroidal involvement in cardiovascular disease.

Evaluating SHAP attributions

  • Directionality: Do high-risk features (older age, hypertension, diabetes) push predictions toward the cardiovascular-disease-related cohorts (FAMILIPO, MRCC)? If age is negatively correlated with MRCC risk, something is wrong.
  • Feature ranking: Do the expected top features appear at the top? Age and hypertension are well-established cardiovascular risk factors and should rank highly for CAD-relevant cohorts.
  • Biomarker contribution: Do OCTA biomarkers add value beyond traditional risk factors? If vessel density and flow-deficit features consistently have near-zero SHAP values, the mask pipeline may not be extracting clinically useful information.
  • Spurious correlations: If a clinically unrelated feature (e.g., sex, when cardiovascular risk is known to be independent of sex after adjusting for other factors) appears at the top of the SHAP ranking, it warrants investigation.

When to be concerned

Red flags that suggest the model is not learning clinically meaningful features:
  • Grad-CAM consistently highlights image borders, corners, or known artifacts.
  • Layer-dominance scores are nearly identical across all four cohorts.
  • SHAP values show a single feature dominating all predictions (e.g., the model is essentially just thresholding age).
  • The integrated analysis shows no correlation between spatial heatmaps and SHAP attributions that should be related (e.g., high CC flow-deficit SHAP but no CC heatmap activation).
Yellow flags that warrant further investigation:
  • A single cohort has very different explainability patterns from the others (may be a data quality issue in that cohort).
  • Misclassified patients have systematically different heatmap patterns from correctly classified ones (may reveal a subphenotype the model cannot distinguish).
  • OCTA biomarkers have high SHAP values that cannot be verified against the raw images (may be artifacts of the mask generation pipeline rather than true biological signal).

Limitations & Caveats

Every explainability method has blind spots. Understanding them is essential for honest interpretation.

Grad-CAM is localization, not segmentation

Grad-CAM produces coarse heatmaps at the resolution of the final feature map (7×7 for ConvNeXt-Tiny, upsampled to 224×224). It cannot resolve fine vascular detail, and the heatmap is a saliency map, not a segmentation mask. Bright regions indicate "important for the decision," not necessarily "contains pathology."

Attention is not explanation

Following the well-known critique by Jain & Wallace (2019) and others, attention weights are a correlational signal, not a causal one. A layer may have high attention weight because its features are easy to attend to, not because they drive the prediction. The Phase 15 layer-ablation experiments are the stronger causal test.

SHAP is approximate and background-dependent

KernelSHAP is an approximation that samples feature subsets. The results depend on the choice of background distribution (the reference dataset used to represent "feature absent"). SHAP also assumes feature independence, which clinical features violate (age and hypertension are correlated, BMI and diabetes are correlated). Correlated features can have their shared importance arbitrarily divided between them.

Model confidence is not model accuracy

A high-confidence prediction with a clean Grad-CAM heatmap can still be wrong. The temperature-scaled probabilities indicate confidence on the training distribution but are not calibrated for out-of-distribution inputs. MC-Dropout entropy provides uncertainty quantification, but uncertainty ≠ error.

Multiple comparision in feature attribution

When examining SHAP values for N clinical features across K cohorts, the probability of finding a "significant-looking" feature by chance increases. SHAP analysis should be treated as hypothesis-generating, and any interesting findings should be validated with hold-out data or independent studies.

Integrated analysis inherits all limitations

The integrated analysis compounds the limitations of each individual method. If Grad-CAM and SHAP disagree, determining which one is "correct" requires external validation (e.g., clinical expert review, ablation experiments). Consistency between methods is encouraging but not proof of correctness.

Best practice: Explainability is for hypothesis generation and trust building, not for causal proof. Strong gradient signals, consistent attention patterns, and plausible SHAP values together form a body of evidence that the model has learned something meaningful. But the gold standard remains prospective clinical validation on independent datasets.
