This version is meant for explaining the project at the right altitude: enough technical substance to defend the modeling choices, but simple enough to describe as one coherent pipeline rather than a collection of notebook phases.

Core idea

The system treats each retinal OCTA layer as a different physiological view, learns a separate visual representation for each one, combines those views with clinical metadata and generated vascular biomarkers, then produces a calibrated coronary artery disease prediction with interpretation hooks.

Inputs Three OCTA layers, per-eye clinical metadata, generated pseudo-mask biomarkers
Fusion Cross-layer attention first, multimodal cross-attention second
Outputs Disease probabilities, calibration, attention attribution, biomarker context

How to explain it

A concise guide for talking through the architecture with a supervisor or collaborator. Each row states the design choice, then the scientific reason it exists.

Why three image streams

The superficial vascular complex, deep vascular complex, and choriocapillaris are not interchangeable pictures of the same thing. They reflect different anatomical compartments and different microvascular failure modes.

Because of that, the model does not force one shared encoder across all layers. Each stream learns its own texture vocabulary before any fusion happens.

Why self-supervised pretraining

Natural-image pretraining is a weak starting point for grayscale OCTA. SimCLR is used to adapt the backbones to retinal vascular structure before supervised disease classification begins.

The bilateral OD/OS pairing is especially useful because both eyes of the same patient share systemic disease context while still providing non-identical anatomy.

Why tabular fusion matters

Image findings are not clinically interpreted in isolation. The same DVC rarefaction pattern means something different in a diabetic patient than in a non-diabetic patient.

The tabular branch therefore acts as context, not as an afterthought. Clinical features and OCTA-derived biomarkers are encoded, then used to condition the visual decision state.

Why generated masks are included

The dataset does not provide manual vessel or FAZ annotations, so the pipeline generates deterministic pseudo-masks and extracts vascular biomarkers from them.

That gives the classifier two complementary signals: dense deep features from the raw images, and more structured morphology-oriented evidence from the pseudo-mask branch.

Why attention appears twice

The first attention block is intra-image: it models how the three retinal layers interact. The second attention block is inter-modality: it lets the fused OCTA state query clinical context selectively.

This separation keeps the biological question and the multimodal question distinct, which makes the architecture easier to justify and easier to interpret.

Why calibration and explainability are explicit

For a clinical screening model, probability quality matters, not just ranking accuracy. The roadmap therefore includes post-training calibration and uncertainty estimation as first-class outputs.

Layer dominance, Grad-CAM maps, biomarker attribution, and ablation studies provide a path to explaining whether the model is using plausible retinal evidence rather than shortcut correlations.
