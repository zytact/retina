Phase 10

Loss functions

The disease classifier uses a primary classification loss with optional augmentation from a segmentation student loss. The default is weighted cross-entropy with label smoothing; Focal Loss is activated when imbalance exceeds a threshold.

Primary classification loss

Weighted cross-entropy with label smoothing is the default. The choice between CE and Focal depends on the measured class imbalance in each training fold.

● Weighted CE + label smoothing

L_cls = -Σ w_k · y_k · log(p_k)

Class weights w_k are set to inverse class frequency, computed per training fold. Label smoothing with ε = 0.1 prevents the model from becoming overconfident and improves calibration.

This is the default loss for all Phase 12 training. It is used whenever the imbalance ratio (max class freq ÷ min class freq) is 10 or less.

● Focal Loss (fallback)

L_focal = -Σ(1 - p_k)^γ · log(p_k)

γ = 2 by default. If any fold's imbalance ratio exceeds 10 ×, the system switches to Focal Loss for that fold. Focal down-weights well-classified samples, focusing gradient signal on the minority classes.

Both losses are compared in experiments. The per-fold choice is logged and reported alongside all metrics.

Class-weighting strategy

The weighting strategy avoids the common mistake of double-correcting for minority classes. Loss weighting and weighted sampling are never used simultaneously.

Default: loss weighting, standard shuffling

Inverse-frequency class weights are applied to the cross-entropy term. Training uses ordinary random shuffling (no WeightedRandomSampler). This single correction is sufficient for imbalance ratios up to 10×.

Ablation: WeightedRandomSampler

If minority-class recall remains poor after the default strategy, a WeightedRandomSampler ablation is tested. In this mode, class weights are removed from the loss to avoid overcorrection. This ablation is explicitly labelled and reported separately — it is never the default.

Segmentation student loss

The upstream U-Net++ student is trained separately against generated pseudo-labels before the disease classifier is trained. Its loss is not included in the downstream classifier objective by default.

● Student composite loss

L_student = α·L_Tversky + (1-α)·L_Focal + λ_dice·L_Dice

Tversky loss with β = 0.7 penalises false negatives more heavily than false positives — critical for sparse vessel, FAZ, and flow-deficit masks. Focal loss (γ = 2) handles class imbalance in the mask pixels. Dice loss provides a directly optimisable overlap metric.

The student is trained against QC-passed classical pseudo-masks, not raw ground truth. During disease-classifier training, the student remains frozen — classifier gradients never backpropagate into the segmenter.

Joint training (optional experiment)

An optional variant unfreezes the segmentation student during Phase 12 Stage 3 fine-tuning and adds the student loss to the total classifier objective: L_total_joint = L_cls + λ₁·L_student. Initial λ₁ = 0.1–0.5, tuned via validation. This variant requires active mask QC monitoring to ensure classifier gradients do not degrade mask quality. It is reported as a separate experiment, not as part of the default model.

Total classifier objective

By default, the disease classifier trains with only the primary classification loss. All other losses are compartmentalised to their respective training stages.

Default

L_total = L_cls (weighted CE with label smoothing, or Focal if imbalanced). This loss backpropagates through every module in the classifier path: OCTA encoders (when unfrozen), projection heads, cross-layer attention, tabular encoder, mask encoder, fusion module, prototype head, and the classification MLP. The segmentation student is excluded.

What is not in L_total

Segmentation student loss: trained upstream, frozen during classifier training.

Calibration loss: temperature scaling is post-hoc only. Differentiable ECE is an optional ablation.

Prototype loss: prototypes are updated by EMA, not by gradient descent.
