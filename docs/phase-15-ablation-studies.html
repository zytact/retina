Phase 15 — Evidence

What Actually Matters?

The full model has many components: three image encoders, a cross-layer attention module, a tabular encoder, a mask encoder, SimCLR pretraining, prototype augmentation, temperature calibration, and more. Are all of these actually pulling their weight? Ablation studies answer this by removing one component at a time and measuring the performance drop.

The Core Question

"If we remove component X, does performance meaningfully decrease?" If the answer is yes, X contributes. If no, X might be unnecessary complexity. Every claim in the paper about "our novel X improves performance" must be backed by an ablation experiment.

Why Ablation Studies Matter for Your Paper

Imagine you have built a 15-component system and it achieves 0.92 AUC. A reviewer asks: "What is the marginal contribution of the cross-layer attention module?" If you cannot answer with a number, your claim that cross-layer attention "improves" performance is speculation, not evidence.

Ablation studies are the standard method for attribution in machine learning research. You systematically remove (or "ablate") each component and retrain from scratch with identical seeds. The performance difference between the full model and the ablated model is the marginal contribution of that component. This is distinct from simply turning off a module at inference time -- you must retrain, because other components may compensate for the missing module during training.

For beginners: Think of it like cooking. You make a complex dish. To know whether the paprika matters, you make the same dish again without paprika. If nobody notices the difference, maybe skip the paprika next time. Ablation is the same idea: make the model again without component X and compare the results.

The Ablation Protocol

Every ablation experiment follows a strict protocol to ensure fair comparison:

  • Identical seeds: Every ablation uses the same random seeds as the full model, so differences are not due to random variation in weight initialization or data ordering.
  • Full retraining: Each ablation is trained from scratch (or from ImageNet weights, depending on the ablation). You cannot just disable a module at test time -- other modules may have compensated during training.
  • Same cross-validation folds: The fixed Phase 2 folds are used everywhere. No re-splitting.
  • Same training schedule: Same epochs, learning rates, optimizer, augmentation, and early stopping. The only thing that changes is the architecture.
  • Mean + std across 5 folds: Every result is reported as mean +/- standard deviation over the 5 folds, with 95% bootstrap confidence intervals.
Critical rule: When you remove a component, the remaining components must still form a valid architecture. For example, removing the tabular branch means the classifier receives only the OCTA and mask embeddings. You cannot leave a broken input dimension.

The Ablation Experiments, Explained

Architecture Ablations (What to remove)

− SimCLR Pretraining

Instead of loading Phase 3 SimCLR-trained backbones, the OCTA encoders are initialized from ImageNet weights only. This tests how much the OCTA-specific self-supervised pretraining contributes. If the performance drop is large, it justifies the 100-epoch SimCLR pretraining stage in the paper.

− Cross-Layer Attention

Instead of passing the three layer tokens (Sup, Deep, CC) through self-attention, we simply concatenate them and project to 256 dimensions. This tests whether the cross-layer interaction modeling is valuable, or whether simply stacking layer features is enough.

− Layer-Identity Embeddings

Remove the learnable e_sup, e_deep, e_cc vectors from the cross-layer attention module. This tests whether the tokens need to "know" which layer they represent, or whether the attention module can infer it from the feature content alone.

− Tabular Branch

Remove all clinical features and generated biomarkers. The classifier runs on OCTA and mask features only. This is a pure imaging model, testing how much the clinical data contributes to classification.

− Mask Branch

Remove the generated segmentation masks and their encoder. Keep the clinical tabular branch. This tests whether the pseudo-masks from Phase 8 add value beyond what the image encoders already extract and what the clinical data already provides.

− Mask Biomarkers

Keep the mask encoder (mask visual features) but remove the quantitative biomarker vector from the tabular stream. This separates the contribution of mask images vs. mask-derived numbers (vessel density, FAZ area, etc.).

Head and Calibration Ablations

− Prototype Head

Remove the prototype-augmented logits, using only the standard linear classifier head. This tests whether the hybrid parametric + non-parametric head helps rare classes.

− Uncertainty (MC-Dropout)

Use a single deterministic forward pass instead of 30-pass MC-Dropout. This tests whether MC-Dropout improves accuracy (by averaging) or only uncertainty estimation.

− Temperature Calibration

Use raw softmax probabilities without post-hoc temperature scaling. This tests whether calibration actually improves probability estimates.

− Label Smoothing

Use hard one-hot targets instead of smoothed targets (epsilon=0). Tests whether label smoothing helps generalization.

Single-Layer and Modality Ablations

These are the most important for the paper's central claim: that using all three layers + clinical data + masks is better than any single modality.

Sup Only / Deep Only / CC Only

Use only one retinal layer. No cross-layer attention (there is only one token). No tabular, no masks. This establishes a per-layer performance baseline. CC-only is expected to be worst (lower resolution, less structured texture), but the actual ordering is an empirical finding.

Sup + Deep

Two encoders, no CC. Tests whether the choriocapillaris layer adds discriminative information beyond the two vascular layers.

Images + Tabular

Three OCTA encoders + clinical features, no mask branch. Tests the marginal contribution of mask features on top of images + clinical data.

Mask Source Ablations

Classical Masks Only

Use the improved classical pseudo-masks (from Phase 8 Step A/B) instead of the U-Net++ student masks. This tests whether the refinement step matters.

Student Masks Only

Use frozen U-Net++ student outputs. The operational default. Compare against classical-only to justify the extra complexity of training a segmentation student.

How to Read an Ablation Table

The ablation results are typically presented as a table in the paper. Each row is an ablation configuration. The columns show metrics (macro-AUC, macro-F1, balanced accuracy, ECE). The full model is the first row (the reference). Each subsequent row shows the performance after removing one component.

When reading the table:

  • A large drop (e.g., -0.05 AUC) means that component is important.
  • A small drop (within the confidence interval) suggests the component may be unnecessary or redundant.
  • A tie or improvement after removal is interesting -- it might mean the component was harming performance (overfitting, noise). This would motivate a simplification of the architecture.
  • Single-layer ablations are expected to show significant drops, confirming the value of multi-layer fusion.

Glossary for Beginners

Ablation
From the Latin "ablatio" (removal). In ML research, systematically removing a component to measure its contribution.
Marginal Contribution
The change in performance when adding or removing a single component, holding everything else constant.
Confidence Interval
A range of values that likely contains the true performance. In our study, 95% bootstrap CIs mean we are 95% confident the true metric lies in that interval.
Bootstrap
A statistical method where you repeatedly resample your data (with replacement) to estimate the variability of a metric. We use patient-cluster bootstrap to account for bilateral eyes.
Parametric vs. Non-Parametric
A parametric model has a fixed set of parameters (like a linear layer). A non-parametric model like prototype classification uses training examples (prototypes) as part of its decision rule. The prototype head combines both approaches.
