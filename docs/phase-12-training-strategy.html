Phase 12

Training strategy

A 3-stage training schedule that first adapts OCTA representations via SimCLR, then trains the downstream classifier modules with frozen backbones, and finally fine-tunes end-to-end. 5-fold patient-grouped cross-validation with inner validation splits for early stopping and calibration.

3-stage training schedule

The three stages progressively unlock model capacity. Stage 1 is a completed reuse, not a rerun. Stages 2 and 3 run sequentially for each fold.

● Stage 1 · OCTA-SimCLR pretraining

Unlabeled adaptation of image encoders to the OCTA domain
Epochs: 100 LR: 0.03 (cosine decay) Batch: 256 (128 × 2 GA) AMP: disabled Grad clip: max norm 1.0

Status: completed and reused. Stage 1 does not rerun pretraining. The notebook loads existing Phase 3 backbone checkpoints and reports a reuse audit. Each fold has 100-epoch histories for Sup, Deep, and CC encoders independently.

● Stage 2 · Frozen OCTA backbones, train downstream heads

Initialise the classifier, attention, fusion, and encoder modules before fine-tuning risk
Epochs: 15 LR: 3e-4 (all trained modules) Frozen: Sup/Deep/CC backbones Frozen: U-Net++ student

Trains the projection heads, tabular encoder, mask encoder, cross-layer attention, multimodal fusion, prototype parameters, and classification head while the OCTA ConvNeXt-Tiny backbones remain frozen. This prevents catastrophic forgetting of the SimCLR-adapted features.

● Stage 3 · End-to-end fine-tuning

Unfreeze OCTA backbones with 10× lower learning rate
Epochs: up to 100 Backbone LR: 3e-5 Head LR: 3e-4 Warmup: 5 epochs (linear) Schedule: cosine warm restarts (T_0=20, T_mult=2) Early stopping: patience 15 on val macro-F1

Unfreezes the OCTA ConvNeXt-Tiny backbones at a 10× lower learning rate than the downstream heads. The U-Net++ student remains frozen throughout. Gradient clipping (max norm 1.0), AMP (bfloat16/float16 with GradScaler), and AdamW are applied.

Cross-validation design

5-fold stratified cross-validation with strict patient-level isolation. Each fold has its own inner validation split for model selection and calibration.

Patient grouping

Both eyes (OD and OS) from the same patient must stay in the same fold. The initial fold assignment was computed during Phase 2 using stratification by cohort label and age decade. Zero patient leakage across folds was verified. These fixed folds are used for all downstream phases — changing them would invalidate the Phase 2, 3, and 8 artifacts.

Inner validation split

For each fold k, the remaining four folds are used as training data. From those four folds, a patient-grouped stratified inner split (80/20 by patient) is carved for early stopping and temperature scaling. The outer fold k is never used for model selection — it is a held-out evaluation set. Final metrics are reported as mean ± std across the 5 outer fold predictions, with bootstrap confidence intervals.

Optimizer and learning rate

AdamW with cosine warm restarts provides the optimisation backbone. Mixed precision reduces memory usage on the GPU training machine.

AdamW

β₁ = 0.9, β₂ = 0.999, weight decay = 1e-2. The weight decay is applied only to non-bias, non-norm parameters.

Learning rate schedule

Stage 2 uses a fixed LR of 3e-4. Stage 3 uses linear warmup for 5 epochs from 0 to the target LRs, then cosine annealing with warm restarts (T_0 = 20 epochs, T_mult = 2). Each restart cycle allows the model to escape sharp minima while converging to flatter solutions that generalise better.

Mixed precision

bfloat16 when available, otherwise float16 with GradScaler. In the notebook, torch.amp.GradScaler('cuda') is used. SimCLR pretraining (Stage 1) had AMP disabled due to gradient instability at LR=0.1, but downstream Stages 2 and 3 use AMP normally.

Batch size

Effective batch size of 32 (physical batch 16 with gradient accumulation ×2 if GPU memory limited). DataLoader pin_memory is disabled by default to avoid CUDA OOM on the Windows GPU machine.

Automatic resumability

Every training stage saves checkpoints every epoch and can resume from the last checkpoint, restoring model weights, optimizer state, RNG state, and training history.

Checkpoint structure

Each fold saves stage*_last.pt (every epoch), stage*_best.pt (best inner-val macro-F1), and stage*_history.csv (per-epoch metrics). The last checkpoint includes optimizer state, AMP scaler state, RNG states (Python, NumPy, PyTorch, CUDA), epoch counter, and early-stopping metadata (best_macro_f1, stale_epochs, early_stopped).

When resumed, the system detects existing checkpoints, loads the last state, and continues from the next epoch. Completed or early-stopped runs are automatically skipped. This is essential for the GPU training machine where runs may span multiple sessions.

Reproducibility

All random seeds are fixed at the start of each fold run. Deterministic PyTorch operations are enabled where available.

Seeds

Python random seed, NumPy seed, PyTorch seed, and CUDA seed are all set to the same value at the start of each fold's Stage 2 launch. The seed value is logged in the fold summary. When resuming, the RNG state is restored from the checkpoint rather than re-seeded, ensuring bitwise-identical continuation.

Deterministic ops

torch.backends.cudnn.deterministic = True and torch.backends.cudnn.benchmark = False are set. This may reduce training throughput slightly but ensures that reruns with the same seed produce identical results, which is critical for ablation study validity.
